Module 01

What is Logistic Regression?

Despite its name, logistic regression is a classification algorithm, not a regression one. It predicts the probability that an instance belongs to a particular class — typically a binary outcome like pass/fail, spam/not-spam, or malignant/benign.

From Linear to Logistic

Linear regression predicts continuous values and can output anything from −∞ to +∞. But probabilities must be between 0 and 1. Logistic regression solves this by wrapping a linear equation inside the sigmoid function, which squashes any real number into the (0, 1) range.

Linear model: z = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ
Sigmoid: σ(z) = 1 / (1 + e⁻ᶻ)
Prediction: P(y=1 | x) = σ(β₀ + β₁x₁ + ... + βₙxₙ)

Key Properties

Parametric — unlike KNN, logistic regression learns a fixed set of weights (β values) during training. Prediction is fast: just a dot product + sigmoid.

Discriminative — it models the decision boundary directly, rather than the full data distribution.

Linear decision boundary — in the feature space, the boundary where P(y=1) = 0.5 is a straight line (or hyperplane in higher dimensions).

Logistic Regression vs. Linear Regression

Linear regression: Predicts a continuous value. Loss = mean squared error. Output ∈ (−∞, +∞).

Logistic regression: Predicts a probability → class. Loss = log loss (binary cross-entropy). Output ∈ (0, 1).

Calculating Probabilities with Logistic Regression

At the heart of logistic regression is the sigmoid function. It takes any linear combination z and maps it to a probability between 0 and 1.

σ(z) = 1 / (1 + e⁻ᶻ)

When z = 0 → σ = 0.5  |  z → +∞ → σ → 1  |  z → −∞ → σ → 0
Hover over the curve to see σ(z) values. Use the sliders to see how β₀ (intercept) and β₁ (slope) change the curve.

Step-by-Step Probability Calculation

Suppose we have a single feature x (hours studied) and want to predict P(pass).

Given: β₀ = −4, β₁ = 1.5

For a student who studied 5 hours:

z = β₀ + β₁ · x = −4 + 1.5 · 5 = 3.5
P(pass) = 1 / (1 + e⁻³·⁵) = 1 / (1 + 0.0302) = 0.9706

The model predicts a 97.1% chance of passing.

Module 03

Using Logistic Regression to Classify

Logistic regression outputs a probability, but classification requires a discrete label. We convert probabilities to classes using a decision threshold — typically 0.5:

ŷ = 1   if P(y=1 | x) ≥ 0.5
ŷ = 0   if P(y=1 | x) < 0.5

The Decision Boundary

The threshold P = 0.5 corresponds to z = 0 (since σ(0) = 0.5). The decision boundary is the set of points where the linear equation equals zero:

β₀ + β₁x₁ + β₂x₂ + ... = 0

With one feature, this is a single point on the x-axis. With two features, it's a line. With more, it's a hyperplane.

Click to add data points (left-click = Class 0, right-click = Class 1). The decision boundary updates automatically. Drag the threshold slider to see how it changes predictions.

Why Adjust the Threshold?

The 0.5 threshold isn't always ideal. In medical screening, you may lower the threshold (e.g., 0.3) to catch more positive cases — accepting more false positives to reduce false negatives. This is the precision-recall tradeoff.

Log Loss (Binary Cross-Entropy)

Logistic regression is trained by minimizing log loss, not mean squared error:

L = −(1/n) × Σ [ yᵢ · log(p̂ᵢ) + (1 − yᵢ) · log(1 − p̂ᵢ) ]

This penalizes confident wrong predictions heavily. If the model says P = 0.99 but the true label is 0, the loss is very large.

Module 04

Expanding to Multiple Input Features

Real-world datasets rarely have just one feature. Logistic regression extends naturally to multiple inputs — each feature gets its own coefficient:

z = β₀ + β₁x₁ + β₂x₂ + β₃x₃ + ... + βₙxₙ
P(y=1) = σ(z) = 1 / (1 + e⁻ᶻ)

Interpreting Multiple Coefficients

Each coefficient βᵢ tells you the effect of feature xᵢ on the log-odds of the positive class, holding other features constant:

log(p / (1−p)) = β₀ + β₁x₁ + β₂x₂ + ...

Positive βᵢ: increasing xᵢ increases the probability of class 1.

Negative βᵢ: increasing xᵢ decreases the probability of class 1.

Magnitude: larger |βᵢ| means feature xᵢ has a stronger influence.

Feature Scaling Still Matters

Just like KNN, logistic regression benefits from feature scaling. While the model will still converge without it, scaling helps the optimization algorithm (gradient descent) converge faster and makes coefficients directly comparable in magnitude.

Key Parameters

C — inverse of regularization strength. Smaller C = stronger regularization = simpler model. Default is 1.0.

penalty — regularization type: 'l2' (Ridge, default), 'l1' (Lasso), or 'none'.

max_iter — maximum iterations for the solver. Increase if you see convergence warnings (e.g., max_iter=1000).

solver — optimization algorithm. 'lbfgs' (default) works well for most cases.

Multiclass Classification

scikit-learn's LogisticRegression handles multiclass automatically using a one-vs-rest (OvR) strategy by default: it trains one binary classifier per class.

Python # Works with 3+ classes automatically model = LogisticRegression(multi_class='ovr', max_iter=1000) model.fit(X_train, y_train) # y has 3+ unique values # predict_proba now returns one column per class probs = model.predict_proba(X_test) # shape: (n_samples, n_classes)
