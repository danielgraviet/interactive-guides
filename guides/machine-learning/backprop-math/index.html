Backpropagation

The chain rule, applied systematically to neural networks.

The one-sentence summary: Backprop computes how much each weight in a network contributed to the error, so we can nudge every weight in the direction that reduces the loss.

1 — The Forward Pass

Every neuron does two things. First, it computes a weighted sum of its inputs (the pre-activation):

z = w₁·x₁ + w₂·x₂ + … + b

Then it pushes that through an activation function (the activation):

a = σ(z)

You chain these layer by layer, left to right, until you reach the output. At the output, you compare to the target using a loss function.

2 — The Loss Function

The loss tells you "how wrong" the network is. Common losses:

Squared error: L = ½(y − ŷ)²

Cross-entropy: L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]

3 — The Backward Pass (Backprop)

Now we reverse direction. Starting at the loss, we ask: "If I wiggle this weight a tiny amount, how much does the loss change?" That's a partial derivative, and we compute it with the chain rule.

The Chain Rule Pattern

At every node, the gradient decomposes into:

∂L/∂w = (upstream gradient) × (local derivative)

The upstream gradient is ∂L/∂z — how the loss changes w.r.t. this node's output. The local derivative is how this node's output changes w.r.t. the weight in question.

Concretely:

To find the gradient for weight w feeding into a neuron:

∂L/∂w = ∂L/∂a · ∂a/∂z · ∂z/∂w

Where:

  • ∂L/∂a — how the loss changes w.r.t. this neuron's activation (comes from the layer above)
  • ∂a/∂z — derivative of the activation function (e.g., σ'(z))
  • ∂z/∂w — this is just the input to that weight!

4 — Weight Update

Once you have all the gradients, update every weight:

w_new = w_old − η · ∂L/∂w

Where η is the learning rate. That's one full step of gradient descent.

Practice Problems

Compute values yourself. Check your answers instantly.

Problem 1 Single Neuron Forward Pass

Given: x = 2, w = −0.5, b = 1, sigmoid activation.

Compute z and a = σ(z).

z = (−0.5)(2) + 1 = 0. Then σ(0) = 1/(1+e⁰) = 1/2 = 0.5

Problem 2 Gradient Computation

Same neuron. Target y = 1, loss L = ½(y − a)².

Using your values from Problem 1 (z = 0, a = 0.5), compute:

∂L/∂a = −(1 − 0.5) = −0.5.   σ'(0) = 0.5·0.5 = 0.25.   ∂z/∂w = x = 2.   Product: (−0.5)(0.25)(2) = −0.25.

Problem 3 Weight Update

With ∂L/∂w = −0.25 and learning rate η = 1.0, compute the new weight.

Does the new weight make sense? The target was 1, the output was 0.5 — the weight should increase to push the output higher.

w_new = −0.5 − (1.0)(−0.25) = −0.5 + 0.25 = −0.25. The weight moved from −0.5 toward zero (less negative), which will increase the output. ✓

Problem 4 ReLU Neuron

Given: x = 3, w = −1, b = 2, ReLU activation. Target y = 0, loss L = ½(y − a)².

z = (−1)(3) + 2 = −1. ReLU(−1) = 0 (clipped!). Since z < 0, ReLU'(z) = 0, so the entire gradient ∂L/∂w = 0. This is the "dead neuron" — no gradient flows back.

The Backprop Recipe

At every neuron, compute the "delta" (error signal):

δ = (upstream gradient) × σ'(z)

Then the weight gradient is:

∂L/∂w = δ × (input to this weight)

And the bias gradient is:

∂L/∂b = δ

To propagate backward, send to the previous layer:

∂L/∂a_prev = δ × w

Key Identities

σ(0) = 0.5  |  σ'(0) = 0.25  |  σ(−z) = 1 − σ(z)

e⁰ = 1  |  ln(1) = 0  |  ln(e) = 1

Sanity Checks

  • Sign check: If the output is too high, gradients should push weights to make it lower (and vice versa).
  • Dead ReLU: If z ≤ 0, the gradient through ReLU is exactly 0.
  • Sigmoid output range: Always between 0 and 1. If you get something outside, recheck.
  • Gradient magnitude: σ' is at most 0.25, so gradients shrink through sigmoid layers — vanishing gradient.
