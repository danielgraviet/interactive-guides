K-Means Clustering

Partitioning data into k non-overlapping groups by minimizing within-cluster variance.

📌 What Is Clustering?

Clustering is an unsupervised learning task that groups data points so that points within the same group (cluster) are more similar to each other than to points in other groups. There are no labels — the algorithm discovers structure on its own.

⚙️ The K-Means Algorithm

Choose k (the number of clusters) ▶

You must specify k before running. There's no single "correct" value — use the elbow or silhouette method to guide this choice.

Initialize k centroids ▶

Randomly place k centroids in feature space. Standard k-means picks random data points; k-means++ spreads them out to converge faster.

Assign every point to the nearest centroid ▶

Use Euclidean distance. Each point belongs to exactly one cluster — the one whose centroid is closest.

Recompute centroids ▶

The centroid of a cluster is the mean of all points assigned to it: c_j = (1/|C_j|) Σ x_i. This is where the name "k-means" comes from.

Repeat until convergence ▶

Alternate between assignment and update steps until centroids stop moving (or a max iteration limit is hit). Convergence is guaranteed but may be to a local optimum.

📐 Centroid & Inertia Calculations

Cluster Centroid

The centroid of cluster C_j is the component-wise mean of all points in the cluster.

Example: Points (1,2), (3,4), (5,6) → centroid = ((1+3+5)/3, (2+4+6)/3) = (3, 4)

Inertia (WCSS)

Inertia = Σ over all clusters Σ over points in cluster ‖x_i − c_j‖². It's the total within-cluster sum of squares — the objective k-means minimizes.

Exam tip: Inertia always decreases as k increases. When k = n, inertia = 0. That's why we need the elbow method — minimizing inertia alone just gives us k = n.

🎯 Choosing k: Elbow & Silhouette

Elbow Method
Silhouette Method

Plot inertia vs. k. The "elbow" is where adding another cluster gives diminishing returns — inertia stops dropping sharply.

There's no formal rule; you look for the bend in the curve. If the plot is smooth with no clear elbow, try the silhouette method instead.

The silhouette score for a point measures how similar it is to its own cluster vs. the nearest other cluster.

s(i) = (b(i) − a(i)) / max(a(i), b(i))

  • a(i) = mean distance from i to all other points in its cluster
  • b(i) = mean distance from i to all points in the nearest neighboring cluster

Range: −1 to +1. High = well-clustered. Near 0 = overlapping. Negative = probably misassigned.

Pick the k that maximizes the average silhouette score across all points.

🐍 Scikit-learn Implementation

from sklearn.cluster import KMeans from sklearn.metrics import silhouette_score # Fit k-means km = KMeans(n_clusters=3, init='k-means++', n_init=10, random_state=42) km.fit(X) # Key attributes labels = km.labels_ # cluster assignment per point centers = km.cluster_centers_ # centroid coordinates inertia = km.inertia_ # WCSS # Silhouette sil = silhouette_score(X, labels) # avg silhouette

⚠️ Limitations & Improvements

When K-Means Struggles

  • Non-globular clusters — k-means assumes spherical clusters of similar size
  • Varying cluster densities or sizes — small/dense vs. large/sparse
  • High-dimensional data — Euclidean distance loses meaning (curse of dimensionality)
  • Outliers — centroids are means, so outliers pull them away
  • Must choose k in advance

Improvements

  • K-Means++ — smarter initialization that spreads initial centroids (default in sklearn)
  • Mini-Batch K-Means — uses random subsets per iteration; much faster on large datasets
  • Multiple random restarts (n_init) — run several times, keep best inertia

Hierarchical Clustering

Building a tree of clusters — no need to specify k up front.

📌 Core Definitions

Hierarchical clustering creates a nested sequence of partitions, visualised as a dendrogram (tree diagram).

Agglomerative Bottom-up: Start with each point as its own cluster, then merge the two closest clusters repeatedly until one cluster remains.

Divisive Top-down: Start with all points in one cluster, then recursively split. Less common, more expensive.

⚙️ Agglomerative Algorithm Steps

Start: each point is a cluster ▶

With n data points, you begin with n singleton clusters.

Compute pairwise distances between all clusters ▶

Use the chosen linkage criterion (single, complete, centroid, or Ward) to define "distance between clusters."

Merge the two closest clusters ▶

The pair with the smallest inter-cluster distance is combined into one new cluster.

Update the distance matrix ▶

Recompute distances from the new merged cluster to all remaining clusters.

Repeat until one cluster remains ▶

This produces the full dendrogram. Cut it at the desired level to get your final clusters.

🔗 Linkage Methods

LinkageDist Between Clusters A, BCharacter
Singlemin dist between any point in A and any point in BCan find elongated clusters; prone to chaining
Completemax dist between any point in A and any point in BCompact, roughly equal-size clusters; sensitive to outliers
Centroiddist between cluster centroidsSimple but can produce inversions in the dendrogram
Wardincrease in total within-cluster variance after mergingMinimizes WCSS like k-means; tends to produce compact, balanced clusters
Exam tip: Ward linkage is the most similar to k-means in spirit — both minimize variance. Single linkage is the most different — it chains.

🌳 Reading a Dendrogram

A dendrogram is a tree where:

  • Leaves = individual data points
  • The height of a merge = the distance (or dissimilarity) at which two clusters joined
  • To choose k clusters, draw a horizontal cut line. The number of vertical lines it crosses = number of clusters.
  • Place the cut where there's the largest vertical gap (biggest jump in merge distance).

Points connected below the cut line belong to the same cluster.

🐍 Implementation

scikit-learn
scipy.cluster
from sklearn.cluster import AgglomerativeClustering agg = AgglomerativeClustering( n_clusters=3, linkage='ward' # 'single','complete','average' ) labels = agg.fit_predict(X)
from scipy.cluster.hierarchy import linkage, dendrogram, fcluster import matplotlib.pyplot as plt # Build linkage matrix Z = linkage(X, method='ward') # Plot dendrogram dendrogram(Z) plt.show() # Cut to get k=3 clusters labels = fcluster(Z, t=3, criterion='maxclust')

⚖️ Pros & Cons

Advantages

  • No need to specify k in advance — choose from the dendrogram
  • Produces a full hierarchy — useful for exploring structure at multiple granularities
  • Works with any distance metric (not just Euclidean)
  • Deterministic — no random initialization

Disadvantages

  • O(n³) time, O(n²) space — doesn't scale well to large datasets
  • Merges are permanent — cannot undo a bad early merge
  • Sensitive to noise and outliers (especially single linkage → chaining)

Other Clustering Algorithms

DBSCAN, Mean-Shift, and Affinity Propagation — each suited to different clustering goals.

🔵 DBSCAN

DBSCAN = Density-Based Spatial Clustering of Applications with Noise. Clusters are dense regions separated by sparse regions.

Key Parameters

eps (ε) — radius of the neighborhood. min_samples — minimum points in the ε-neighborhood to be a core point.

Point Types

  • Core point: has ≥ min_samples points within ε
  • Border point: within ε of a core point but not itself core
  • Noise point: neither core nor border → label = −1
Pick an unvisited point ▶

Start from any arbitrary unvisited point in the dataset.

Query its ε-neighborhood ▶

Find all points within distance ε. If count ≥ min_samples → it's a core point; start a new cluster.

Expand the cluster ▶

Add all ε-neighbors. For each that is also a core point, recursively add their neighbors. Continue until no more core-reachable points.

Repeat for remaining unvisited points ▶

Any point never reached is labelled noise (−1).

from sklearn.cluster import DBSCAN db = DBSCAN(eps=0.5, min_samples=5) labels = db.fit_predict(X) # Noise points have label == -1

🟠 Mean-Shift Clustering

Mean-Shift moves each point toward the densest area of points within a bandwidth window, finding "peaks" in the density landscape.
Place a window on each data point ▶

Each point gets a kernel window of a fixed bandwidth radius.

Compute mean of points within each window ▶

The "mean shift" vector points from the current center toward the region of higher density.

Shift window center to the mean ▶

Move the kernel to where the mean was computed. This climbs the density gradient.

Repeat until convergence ▶

Points that converge to the same peak are assigned the same cluster.

from sklearn.cluster import MeanShift, estimate_bandwidth bw = estimate_bandwidth(X, quantile=0.2) ms = MeanShift(bandwidth=bw) labels = ms.fit_predict(X)

🔴 Affinity Propagation

Affinity Propagation passes "messages" between data points to find a set of representative exemplars. It discovers k automatically.
Compute similarity matrix ▶

Typically negative squared Euclidean distance: s(i,k) = −‖x_i − x_k‖². Diagonal s(k,k) = preference (controls number of clusters).

Exchange "responsibility" and "availability" messages ▶

Responsibility r(i,k): how well-suited k is as an exemplar for i. Availability a(i,k): how appropriate it would be for i to choose k. Messages iterate back and forth.

Iterate until convergence ▶

Messages are damped to prevent oscillation. Convergence = exemplar assignments stop changing.

Assign each point to its exemplar ▶

Points whose r(k,k) + a(k,k) > 0 become exemplars. Every other point is assigned to its best exemplar.

from sklearn.cluster import AffinityPropagation ap = AffinityPropagation(damping=0.9, random_state=42) labels = ap.fit_predict(X) exemplars = ap.cluster_centers_indices_
