Why Reduce Dimensions?

Dimensionality reduction removes redundant or irrelevant features, fighting the curse of dimensionality, speeding up training, and improving generalization.

Analogy

The Packed Suitcase

Imagine packing for a trip. You could throw in everything you own, but your suitcase would be enormous, hard to carry, and you'd waste time digging for what you need. Dimensionality reduction is like packing only the essentials — you keep what matters and leave behind what's redundant or useless. Feature selection is choosing which items to pack. Feature extraction is rolling all your clothes into compact cubes — new representations that capture the same information in less space.

Filter-Based Feature Selection

Filter methods score each feature independently using statistical tests — no model is trained. They're fast but ignore feature interactions.

Concept

Choosing the Right Statistical Test

The test you use depends on the type of feature (continuous vs. categorical) and the type of target:

Feature ↓ / Target →Continuous TargetCategorical Target
Continuous featurePearson correlation / F-regression (ANOVA)ANOVA F-test / Mutual Information
Categorical featureANOVA F-test (feature groups → target)Chi-squared (χ²) / Mutual Information
f_classif f_regression chi2 mutual_info_classif mutual_info_regression
Analogy

Speed Dating for Features

Filter methods are like speed dating: you evaluate each candidate (feature) one at a time against a simple criterion (statistical score). You don't see how candidates interact as a group — you're just screening individually. Fast, but you might miss that two mediocre features together are excellent predictors.
Concept

Key Statistical Tests Explained

χ² (Chi-Squared) Test▼
Tests whether a categorical feature is independent of a categorical target. High χ² ⇒ the feature and target are dependent ⇒ feature is likely useful. Requires non-negative feature values. Used via sklearn.feature_selection.chi2.
ANOVA F-Test (f_classif / f_regression)▼
Compares the variance between groups to the variance within groups. A high F-statistic means the feature separates target classes well. Assumes linear relationship and normality. f_classif for classification; f_regression for regression targets.
Mutual Information▼
Measures how much knowing the feature reduces uncertainty about the target. Captures nonlinear relationships (unlike Pearson/ANOVA). MI = 0 means independent; higher = more dependent. mutual_info_classif or mutual_info_regression.
Pearson Correlation▼
Measures linear correlation between a continuous feature and continuous target. Range: −1 to +1. Sensitive to outliers. Does NOT capture nonlinear relationships. Often used before f_regression.
Code

Filter Selection with scikit-learn

from sklearn.feature_selection import SelectKBest, f_classif, chi2, mutual_info_classif

# Select top 10 features using ANOVA F-test (classification)
selector = SelectKBest(score_func=f_classif, k=10)
X_new = selector.fit_transform(X, y)
selected_mask = selector.get_support() # Boolean mask of selected features
scores = selector.scores_ # F-statistics for each feature

# Chi-squared (for non-negative categorical features)
selector_chi = SelectKBest(score_func=chi2, k=10)
X_chi = selector_chi.fit_transform(X, y)

# Mutual information (captures nonlinear)
selector_mi = SelectKBest(score_func=mutual_info_classif, k=10)
X_mi = selector_mi.fit_transform(X, y)

Wrapper Methods: RFE & Sequential Selection

Wrapper methods use a model's performance to evaluate feature subsets. More accurate than filters, but computationally expensive.

Steps

Recursive Feature Elimination (RFE)

  1. Train a model on all features.
  2. Rank features by importance (e.g., coefficients, feature importances).
  3. Remove the least important feature(s).
  4. Retrain the model on the remaining features.
  5. Repeat steps 2–4 until the desired number of features is reached.
Think of RFE like a reality show elimination: every round, the weakest performer gets voted off. The model is the panel of judges, retrained after each cut.
Steps

Forward Sequential Feature Selection (SFS)

  1. Start with an empty feature set.
  2. For each remaining feature, tentatively add it and evaluate model performance (e.g., via cross-validation).
  3. Permanently add the feature that gives the best performance improvement.
  4. Repeat until you reach the desired number of features or performance stops improving.
Steps

Backward Sequential Feature Selection

  1. Start with all features included.
  2. For each included feature, tentatively remove it and evaluate model performance.
  3. Permanently remove the feature whose removal causes the least performance drop (or greatest improvement).
  4. Repeat until you reach the desired number of features.
Forward SFS builds a team one player at a time, always adding the best available. Backward SFS starts with the full roster and cuts the least valuable player each round.
Code

Wrapper Methods in scikit-learn

from sklearn.feature_selection import RFE, SequentialFeatureSelector
from sklearn.ensemble import RandomForestClassifier

# ── Recursive Feature Elimination ──
model = RandomForestClassifier(n_estimators=100)
rfe = RFE(estimator=model, n_features_to_select=5, step=1)
X_rfe = rfe.fit_transform(X, y)
print(rfe.support_) # Boolean mask
print(rfe.ranking_) # Feature ranking (1 = selected)

# ── Forward Sequential Selection ──
sfs_fwd = SequentialFeatureSelector(
    model, n_features_to_select=5, direction='forward', cv=5
)
X_fwd = sfs_fwd.fit_transform(X, y)

# ── Backward Sequential Selection ──
sfs_bwd = SequentialFeatureSelector(
    model, n_features_to_select=5, direction='backward', cv=5
)
X_bwd = sfs_bwd.fit_transform(X, y)

Embedded Methods

Embedded methods perform feature selection as part of model training — they get the accuracy benefits of wrappers without the full computational overhead.

Concept

Why Embedded Methods Win

AdvantageOver FiltersOver Wrappers
Feature interactionsCaptures them (filters don't)Same benefit
Computational costComparableMuch cheaper — single training pass
Overfitting riskSimilarLower — no exhaustive subset search
Model-specificityFeature selection is tuned to the model that will actually be usedSame benefit
Concept

Common Embedded Methods

Lasso (L1 Regularization)▼
Adds |w| penalty to the loss function. Drives irrelevant feature weights to exactly zero, performing automatic feature selection. The strength of selection is controlled by the regularization parameter α.
Ridge (L2 Regularization)▼
Adds w² penalty. Shrinks coefficients toward zero but doesn't set them exactly to zero — no true feature elimination. Useful when all features contribute somewhat.
Elastic Net▼
Combines L1 and L2 penalties. Gets sparsity from L1 while handling correlated features better than pure Lasso. Controlled by α (total regularization) and l1_ratio (mix between L1 and L2).
Tree-Based Feature Importance▼
Random Forests and Gradient Boosted Trees compute feature importances during training (e.g., mean decrease in impurity). Use SelectFromModel to keep only features above a threshold. No extra training required.
Code

Embedded Methods in scikit-learn

from sklearn.linear_model import Lasso, LogisticRegression
from sklearn.ensemble import RandomForestClassifier
from sklearn.feature_selection import SelectFromModel

# ── L1 / Lasso (sets coefficients to 0) ──
lasso = Lasso(alpha=0.01)
lasso.fit(X, y)
selected = X.columns[lasso.coef_ != 0]

# ── Logistic Regression with L1 penalty ──
lr = LogisticRegression(penalty='l1', solver='saga', C=0.1)
selector = SelectFromModel(lr)
X_embedded = selector.fit_transform(X, y)

# ── Tree-based importance ──
rf = RandomForestClassifier(n_estimators=200)
selector_tree = SelectFromModel(rf, threshold='median')
X_tree = selector_tree.fit_transform(X, y)

Linear Feature Extraction

PCA, ICA, and Factor Analysis create new features by linearly combining the originals. Each has different assumptions and goals.

Concept

PCA — Principal Component Analysis

Unsupervised Variance-maximizing Orthogonal components

Goal: Find orthogonal axes (principal components) that capture the most variance in the data.

Key terminology:

  • Eigenvalue — Amount of variance explained by a component
  • Eigenvector — Direction of a principal component
  • Loadings — Correlations between original features and components
  • Explained variance ratio — Proportion of total variance captured by each component
  • Scree plot — Plot of eigenvalues to decide how many components to keep

Algorithm steps:

  1. Standardize the data (mean=0, std=1) — PCA is sensitive to scale.
  2. Compute the covariance matrix of the standardized data.
  3. Compute the eigenvalues and eigenvectors of the covariance matrix.
  4. Sort eigenvectors by decreasing eigenvalue.
  5. Select top k eigenvectors — these are your principal components.
  6. Project data onto these k components: X_new = X × W_k
Concept

ICA — Independent Component Analysis

Unsupervised Independence-maximizing Non-Gaussian sources

Goal: Separate a multivariate signal into statistically independent (not just uncorrelated) source signals.

Key terminology:

  • Mixing matrix — How source signals were mixed to produce observations
  • Unmixing matrix — Learned inverse to recover independent sources
  • Non-Gaussianity — Measured via kurtosis or negentropy; ICA maximizes this
  • Cocktail party problem — Classic analogy: separating individual speakers from mixed audio

Algorithm steps (FastICA):

  1. Center the data (subtract mean).
  2. Whiten the data (decorrelate and equalize variance) — often via PCA.
  3. Initialize a random unmixing vector.
  4. Iteratively update the vector to maximize non-Gaussianity (e.g., via negentropy approximation).
  5. Decorrelate the vector from previously found components (deflation or symmetric approach).
  6. Repeat for each desired independent component.
Concept

Factor Analysis (FA)

Unsupervised Latent variable model Assumes noise

Goal: Model observed variables as linear combinations of latent factors plus feature-specific noise.

Model: X = μ + L·F + ε   where L = loadings, F = factors, ε = noise

Key terminology:

  • Factor loadings — How much each observed variable relates to each factor
  • Communality — Proportion of a variable's variance explained by the factors (1 − unique variance)
  • Uniqueness — Variance specific to each variable (noise)
  • Factor rotation — Varimax, Promax — make loadings easier to interpret

Algorithm steps:

  1. Estimate the correlation/covariance matrix.
  2. Extract initial factors (via maximum likelihood or principal axis factoring).
  3. Determine the number of factors (eigenvalues > 1, scree plot, parallel analysis).
  4. Rotate factors for interpretability (e.g., Varimax for orthogonal, Promax for oblique).
  5. Interpret rotated factor loadings — which variables load onto which factors.
  6. Compute factor scores for each observation.

Nonlinear Feature Extraction

When data lies on a curved manifold, linear methods fail. MDS, Isomap, and t-SNE capture nonlinear structure.

Concept

MDS — Multidimensional Scaling

Distance-preserving Unsupervised

Goal: Find a low-dimensional configuration that preserves pairwise distances (or dissimilarities) between points as well as possible.

Key terminology:

  • Stress — Objective function measuring mismatch between high-D and low-D distances
  • Dissimilarity matrix — Input matrix of pairwise distances
  • Metric vs. Non-metric MDS — Metric preserves actual distances; non-metric preserves only rank order

Algorithm steps (metric MDS):

  1. Compute the pairwise distance matrix in the original space.
  2. Initialize points in the low-dimensional space (randomly or via PCA).
  3. Compute distances in the low-D space.
  4. Compute stress — the mismatch between high-D and low-D distances.
  5. Optimize point positions to minimize stress (gradient descent or SMACOF).
  6. Repeat until convergence.
Concept

Isomap

Geodesic distances Graph-based Unsupervised

Goal: Preserve geodesic (along-the-manifold) distances, not straight-line Euclidean distances.

Key terminology:

  • Geodesic distance — Shortest path along the manifold surface (like walking on a hill vs. tunneling through it)
  • k-nearest neighbor graph — Connects each point to its k closest neighbors
  • Shortest path — Dijkstra's or Floyd-Warshall on the graph approximates geodesic distances

Algorithm steps:

  1. Construct a k-nearest neighbor graph — edges weighted by Euclidean distance.
  2. Compute shortest path distances between all pairs using Dijkstra's algorithm. These approximate geodesic distances.
  3. Apply classical MDS to the geodesic distance matrix to obtain the low-D embedding.
Imagine two points on opposite sides of a Swiss roll. Euclidean distance says they're close (straight through the roll). Geodesic distance follows the surface — much longer and more faithful to the true structure. Isomap uses graph shortest paths to approximate this.
Concept

t-SNE (t-Distributed Stochastic Neighbor Embedding)

Probabilistic Local structure Unsupervised

Goal: Preserve local neighborhood structure — points that are close in high-D should be close in low-D. Great for visualization.

Key terminology:

  • Perplexity — Balances local vs. global structure (~5–50 typical). Roughly "effective number of neighbors"
  • KL divergence — Objective: minimize the gap between high-D and low-D probability distributions
  • Student's t-distribution — Used in low-D to allow dissimilar points to spread far apart (heavier tails than Gaussian)
  • Crowding problem — In low-D, you can't maintain all distances; the t-distribution helps alleviate this

Algorithm steps:

  1. Compute pairwise conditional probabilities in high-D using a Gaussian kernel (controlled by perplexity).
  2. Symmetrize: p_ij = (p_j|i + p_i|j) / 2n.
  3. Initialize low-D points (randomly or via PCA).
  4. Compute pairwise probabilities in low-D using a Student's t-distribution with 1 degree of freedom.
  5. Minimize KL divergence between the two distributions via gradient descent.
  6. Iterate until convergence (using early exaggeration and momentum).

PCA Deep Dive

Performing PCA end-to-end: choosing components, interpreting loadings, and comparing PCA-based models to original-feature models.

Concept

How to Select the Number of Components

Three common strategies:

1. Cumulative Explained Variance Threshold▼
Keep components until you reach a threshold (e.g., 90% or 95% of total variance). In sklearn: PCA(n_components=0.95) automatically selects the right number. This is the most common approach for practical ML.
2. Scree Plot / Elbow Method▼
Plot eigenvalues (or explained variance) vs. component number. Look for the "elbow" — the point where adding more components yields diminishing returns. Keep components before the elbow.
3. Kaiser Criterion (eigenvalue > 1)▼
Keep only components whose eigenvalue exceeds 1 (on standardized data). Rationale: a component should explain at least as much variance as a single original feature. Simple but can over- or under-extract.
Concept

Correlation Matrix: Features vs. Components

The loading matrix shows how each original feature correlates with each principal component:

  • High positive loading (+0.7 to +1.0) → feature strongly contributes to the component in the positive direction
  • High negative loading (−0.7 to −1.0) → feature contributes strongly in the opposite direction
  • Near zero → feature has little relationship with that component

How to compute it: loadings = eigenvectors × √eigenvalues. In sklearn: loadings = pca.components_.T * np.sqrt(pca.explained_variance_)

Think of each PC as a "theme" that summarizes a group of features. The correlation matrix tells you which features belong to which theme and how strongly. PC1 might be the "size" theme if height, weight, and BMI all load heavily on it.
Concept

Original Features vs. PCA Components as Model Inputs

A common experiment: train a model on all original features, then train the same model on PCA-reduced features. Compare:

Code

Full PCA Pipeline & Model Comparison

import numpy as np
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score

# Step 1: Standardize
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Step 2: PCA with 95% variance threshold
pca = PCA(n_components=0.95)
X_pca = pca.fit_transform(X_scaled)
print(f"Components kept: {pca.n_components_}")
print(f"Total variance: {sum(pca.explained_variance_ratio_):.2%}")

# Step 3: Loadings (feature-component correlations)
loadings = pca.components_.T * np.sqrt(pca.explained_variance_)
# loadings[i, j] = correlation of feature i with PC j

# Step 4: Compare model performance
model = LogisticRegression(max_iter=1000)

scores_orig = cross_val_score(model, X_scaled, y, cv=5)
scores_pca = cross_val_score(model, X_pca, y, cv=5)

print(f"Original: {scores_orig.mean():.4f} ± {scores_orig.std():.4f}")
print(f"PCA-based: {scores_pca.mean():.4f} ± {scores_pca.std():.4f}")

CFeatures with larger scales would dominate the variance and thus the principal components
DStandardization makes the data Gaussian, which PCA requires
PCA maximizes variance. If one feature is measured in millions and another in fractions, the high-scale feature would dominate PC1 regardless of its importance. Standardizing (mean=0, std=1) puts all features on equal footing. PCA does not require Gaussian data.
Quiz
Q22: In the loading matrix, feature X has loadings [0.85, -0.02, 0.10] on PC1, PC2, PC3. What can you conclude?
AFeature X is strongly associated with PC1 and weakly with PC2 and PC3
BFeature X is equally important across all three components
CFeature X should be removed since it only loads on one component
DFeature X is independent of the data
A loading of 0.85 on PC1 means feature X is highly correlated with PC1 — it's a major contributor to the "theme" that PC1 represents. Near-zero loadings on PC2/PC3 mean it contributes little to those components. This is normal and useful information.
Quiz
Q23: A logistic regression trained on PCA components outperforms one trained on all original features. The most likely reason is:
APCA creates better features from scratch
BPCA removed noise and multicollinearity, reducing overfitting
CPCA is a supervised method that optimizes for the target
DLogistic regression cannot work with more than 10 features
PCA is unsupervised — it doesn't see the target. But by removing low-variance (noisy) directions and multicollinearity, it provides cleaner inputs. With fewer, orthogonal features, the model generalizes better, especially when d is large relative to n.
Quiz
Q24: The Kaiser criterion recommends keeping components with eigenvalues:
AGreater than 0
BGreater than the mean eigenvalue
CGreater than 1 (on standardized data)
DIn the top 50th percentile
On standardized data, each original feature contributes a variance of 1. A component with eigenvalue > 1 explains more variance than any single original feature, making it "worth keeping." This is the Kaiser criterion. Note: on standardized data, the mean eigenvalue IS 1, so B and C are related but C is the standard formulation.

Final Review — Comprehensive MCQs

Professor-style questions that cross topics and test deeper understanding.

Quiz
Q25: You're building a medical diagnosis model where doctors need to understand WHY the model made a prediction. Which approach is best?
AFeature selection (e.g., Lasso) — keeps original interpretable features
BPCA — creates compact components
Ct-SNE — excellent visualization
DICA — separates independent signals
Interpretability is paramount in medicine. Feature selection retains original feature names (e.g., "blood pressure", "age") so doctors can explain decisions. PCA/ICA/t-SNE create abstract components that lack direct medical meaning.
Quiz
Q26: You want to visualize 50,000 document embeddings (300D) to find topical clusters. Best technique?
APCA down to 2D
BIsomap
Ct-SNE (possibly after PCA to ~50D first)
DForward feature selection
t-SNE excels at revealing cluster structure in high-D data for visualization. A common pipeline: PCA to ~50D first (speed + denoising), then t-SNE to 2D. Isomap would be O(n²) — too slow for 50k points. PCA alone often fails to reveal cluster structure in 2D.
Quiz
Q27: Which of these is NOT an advantage of Elastic Net over pure Lasso?
ABetter handling of correlated features (grouping effect)
BMore stable feature selection
CCan select more features than observations (when n < p)
DIt does not require any hyperparameter tuning
Elastic Net has TWO hyperparameters (α and l1_ratio) — more tuning than Lasso (just α). Its advantages are the grouping effect (selecting correlated features together), stability, and working when p > n. Pure Lasso can select at most n features when p > n.
Quiz
Q28: After running PCA, you find that PC1 explains 80% of variance and PC2 explains 15%. When you check loadings, PC1 has high loadings on "height", "weight", and "shoe_size". PC1 likely represents:
AA noise component that should be discarded
BA latent "body size" factor capturing shared variance across size-related features
CAn artifact of not standardizing the data
DThe target variable leaking into the features
When correlated features (all related to body size) load heavily on the same PC, that component captures their shared latent construct — "body size." This is exactly what PCA is designed to find: the main directions of variation in the data.
Quiz
Q29: You run t-SNE on a dataset and see 5 clear clusters. Your colleague argues that Cluster A is "much farther" from Cluster B than from Cluster C based on the plot. Is this valid?
AYes — t-SNE preserves all pairwise distances
BNo — t-SNE only preserves local structure; inter-cluster distances are unreliable
CYes — the t-distribution ensures global distances are accurate
DIt depends on the perplexity setting
t-SNE's KL divergence objective strongly preserves local neighborhoods (which points are NEAR each other) but does not preserve global structure. The distances between clusters in a t-SNE plot are essentially meaningless — you cannot conclude one cluster is "farther" than another.
Quiz
Q30: Which technique produces components that are statistically INDEPENDENT (not just uncorrelated)?
APCA
BICA
CFactor Analysis
DMDS
PCA produces uncorrelated components (zero linear correlation), but ICA goes further — it finds components that are statistically independent (no relationship of any order). This is a stronger condition. Independence ⊃ uncorrelatedness.
Quiz
Q31: You have 20 features and want to select the best 5 using a Random Forest. You want to account for feature interactions but training is cheap. Best approach?
AChi-squared filter
BMutual information filter
CRFE with the Random Forest
DPCA keeping 5 components
With only 20 features and cheap training, RFE is practical and will capture interactions (Random Forest natively models them). Filters evaluate features independently. PCA is extraction, not selection — you'd lose original feature names. RFE + RF is the sweet spot here.
Quiz
Q32: A researcher applies Factor Analysis and finds that a variable has communality = 0.15. This means:
AThe variable is the most important in the dataset
B85% of the variable's variance is explained by the factors
COnly 15% of the variable's variance is explained by the factors — it's mostly noise/unique
DThe variable loads equally on all factors
Communality of 0.15 means the extracted factors explain only 15% of that variable's variance. The remaining 85% is "uniqueness" — specific to that variable or noise. This variable may not fit well with the factor model and could be a candidate for removal.
Quiz
Q33: SelectKBest with f_classif and SelectKBest with mutual_info_classif: when would they disagree most?
AWhen all features are perfectly correlated with the target
BWhen the dataset has very few samples
CWhen all relationships are linear
DWhen features have strong nonlinear relationships with the target
f_classif (ANOVA F-test) only detects linear separability between classes. Mutual information captures any dependency, including nonlinear ones. If a feature has a strong U-shaped or circular relationship with the target, MI would rank it high while f_classif might rank it low.
Quiz
Q34: You need to reduce dimensionality of a dataset for a downstream classifier. You can afford 5 minutes of compute. Features: 500, Samples: 10,000. Rank these approaches from fastest to slowest:
AFilter → PCA → Embedded → Wrapper
BWrapper → Embedded → PCA → Filter
CPCA → Filter → Wrapper → Embedded
DEmbedded → Filter → PCA → Wrapper
Filter methods just compute statistics — O(n·d). PCA is a single eigendecomposition — O(d²·n). Embedded methods train one model — O depends on model but one pass. Wrappers train many models — O(d² × model training). From fastest to slowest: Filter → PCA → Embedded → Wrapper.
