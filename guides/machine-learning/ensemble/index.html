Chapter One

What is an ensemble, really?

Core Idea

An ensemble is a committee of learners. Rather than trust a single model to solve a problem, you train many and combine their predictions — by voting for classification or averaging for regression. The guiding intuition dates back to the 18th-century Condorcet Jury Theorem: if each juror is even slightly better than random, assembling enough of them drives the group's error toward zero.

But there's a catch that makes ensembles interesting rather than trivial: the jurors must disagree with each other in the right way. A room of 1,000 identical clones votes the same as one clone. The magic of ensembles lives in the diversity of their errors, not just their count.

If all your models make the same mistakes, combining them buys you nothing. Ensembles profit only from uncorrelated error.
Chapter Two

The two criteria for ensembling.

Necessary Conditions

Before an ensemble can outperform its best member, two conditions must hold simultaneously. Miss either one, and you're just spending compute.

Criterion 01
Accuracy
Each base learner must be better than random guessing. For a binary classifier, this means error rate < 50%. Learners that do worse than a coin flip are called "weak" (bad sense) and drag the committee down, because their votes correlate the wrong way with truth.
Criterion 02
Diversity
The learners must make different errors on different inputs. If three models all fail on the same examples, a majority vote still fails. Diversity is what averages out individual mistakes and lets the ensemble's error drop below any single member's.
The Payoff
Variance Reduction
When both criteria hold, combining N independent learners with variance σ² reduces variance to roughly σ²/N. That's why bagging works: same bias, lower variance, better generalization. Bias is unchanged — you can't average your way out of a bad model family.

Question 01 of 08
You build a 50-model ensemble. Each model has 55% accuracy on a binary task, but they all make identical predictions. What is the ensemble's accuracy?
Correct — option C. This is the diversity criterion in its starkest form. Condorcet's theorem requires independent errors. When all voters cast the same ballot, the majority vote is just that single ballot repeated. No diversity → no improvement.
Chapter Three

Three families of ensemble.

Taxonomy

Ensembles fall into three canonical families, each with a different philosophy about how the base learners relate to one another. The distinction matters — on an exam, confusing bagging with boosting is the classic trap.

Family How it trains Learner relationship Primary benefit Flagship example
Bagging Learners trained in parallel on bootstrap samples of the data Independent — no knowledge of each other Reduces variance Random Forest
Boosting Learners trained sequentially, each focuses on previous mistakes Dependent — each builds on the last Reduces bias (and variance) AdaBoost, XGBoost
Stacking Diverse learners' outputs fed to a meta-learner Heterogeneous base, learned combiner Captures model-specific strengths Blended Kaggle winners
Question 02 of 08
Which ensemble family is characterized by training learners sequentially, with each new learner focusing on the errors of the previous ones?
Correct — option B, boosting. The word "sequential" is your signal. Bagging trains in parallel with no cross-talk; stacking trains base models independently then learns a combiner. Only boosting has each learner explicitly targeting what the previous ones missed.
Chapter Four

Where does diversity come from?

Sources of Variation

If diversity is the fuel, where do we get it? There are two fundamental strategies — and most real-world ensembles exploit both at once. Understand this distinction and a lot of ensemble design stops feeling ad hoc.

Data
Vary the data.
Give each learner a different slice of the training set. The algorithm stays the same, but because each model sees a different sample, they converge to different hypotheses.
Techniques: bootstrap sampling (bagging), subsampling rows, subsampling features (random subspaces), weighted resampling (boosting).
Algorithm
Vary the algorithm.
Give each learner the same data but a different inductive bias. A decision tree, a k-NN, and an SVM will make systematically different errors on the same examples.
Techniques: heterogeneous base learners (stacking), different hyperparameters, different random seeds, different feature engineering.
Both
The Random Forest trick.
Random Forests combine both. Each tree sees a different bootstrap sample (data variation) and at each split considers only a random subset of features (algorithmic variation via injected randomness). The result: highly decorrelated trees.
Same algorithm, different data. Same data, different algorithm. Or — for the winners — both at once.
Question 03 of 08
A Random Forest uses which sources of variation to create diverse trees?
Correct — option C. This is what separates a Random Forest from a plain bagged ensemble of trees. The bootstrap handles data variation; the random-subset-per-split rule injects algorithmic variation. This combination is what makes RF famously robust.
Chapter Five

Bootstrapping & bagging.

Definition

Bootstrapping is sampling with replacement from a dataset of size N to produce a new dataset of size N. Because of replacement, some original points appear multiple times and others don't appear at all. On average, each bootstrap sample contains about 63.2% of the unique original points — the rest are duplicates.

Bagging (Bootstrap AGGregatING) applies this trick: train one model per bootstrap sample, then aggregate their predictions by voting or averaging. The samples differ, so the models differ, so their errors partially decorrelate — and averaging decorrelated estimators reduces variance.

The leftover ~36.8% of points each model never saw is called its out-of-bag (OOB) set — a free validation set, no cross-validation required.

Question 04 of 08
You have a training set with 1,000 examples. You draw a bootstrap sample. Approximately how many unique original examples will it contain?
Correct — option B. The probability that any given point is never drawn in N tries is (1 − 1/N)N, which converges to 1/e ≈ 36.8% as N grows. So the expected fraction drawn at least once is 1 − 1/e ≈ 63.2%. Option D describes the OOB set — the complement.
Question 05 of 08
Bagging is most effective when the base learner is:
Correct — option B. Bagging reduces variance but not bias. So it pays off when the base learner has plenty of variance to trim — which is exactly what unpruned trees, deep neural nets, and other flexible estimators offer. Bagging a linear regression gives you essentially the same linear regression.
Chapter Six

Boosting: AdaBoost & gradient.

The Sequential Idea

Boosting flips bagging's philosophy. Instead of training independent models in parallel, boosting trains them one after another, with each new model explicitly targeting the examples its predecessors got wrong. The final prediction is a weighted combination — better-performing learners get louder votes.

Boosting can turn a set of genuinely weak learners — decision stumps barely better than coin flips — into an arbitrarily strong committee. This was a breakthrough result: it showed that "weak learnability" and "strong learnability" are mathematically equivalent (Schapire, 1990).

AdaBoost — reweight the hard examples.

Algorithm

AdaBoost (Adaptive Boosting) works by maintaining a weight on each training example. Initially all weights are equal. After training a weak learner:

1. Compute the learner's weighted error ε.
2. Give the learner a vote weight α = ½ ln((1−ε)/ε). Lower error → larger α.
3. Increase weights on misclassified examples, decrease weights on correctly classified ones.
4. Normalize weights, and repeat.

Examples the current committee keeps getting wrong accumulate weight and increasingly dominate what the next weak learner optimizes. The final prediction is sign(Σ αt ht(x)).

Gradient Boosting — fit the residuals.

Algorithm

Gradient Boosting generalizes the same idea to any differentiable loss function. Instead of reweighting examples, it fits each new learner to the residual errors (or more precisely, the negative gradient of the loss) of the current ensemble.

1. Start with a constant prediction F0(x) = mean(y).
2. Compute residuals ri = yi − F(xi) — the direction the prediction should move.
3. Train a new weak learner h to predict those residuals.
4. Update: F(x) ← F(x) + η · h(x), where η is the learning rate.
5. Repeat for T rounds.

This is literally gradient descent in function space. AdaBoost is the special case when the loss is exponential and the base learner is a classifier. XGBoost, LightGBM, and CatBoost are gradient boosting implementations with engineering tweaks.

Question 06 of 08
In AdaBoost, a weak learner achieves a weighted error of ε = 0.1. A second weak learner achieves ε = 0.4. Which gets the larger vote weight α in the final ensemble?
Correct — option A. The formula α = ½ ln((1−ε)/ε) is monotonically decreasing in ε. For ε=0.1, α ≈ 1.10. For ε=0.4, α ≈ 0.20. Better learners shout louder in the final vote. (A learner with ε=0.5 gets α=0, exactly what a random guesser deserves.)
Question 07 of 08
What does each new learner in a gradient boosting regressor try to predict?
Correct — option C. Option B describes AdaBoost; option D describes bagging. Gradient boosting's signature move is fitting each new learner to what the current ensemble got wrong — the residual. For squared-error loss, the negative gradient equals the residual y − F(x), which is where the intuition comes from.

Final self-check.

Question 08 of 08 — synthesis
A classmate says: "I bagged 100 linear regressions with bootstrap samples from my dataset, but the ensemble has nearly the same accuracy as a single linear regression on the full data." Why?
Correct — option B. This ties together criterion and source of variation. Bagging reduces variance but cannot reduce bias. Linear regression is a high-bias, low-variance model — there's barely any variance for bagging to trim. To benefit from bagging, use a high-variance base learner like a deep tree. This is exactly why Random Forests use fully-grown, unpruned trees: maximum variance, maximum room for bagging to help.
