Chapter One

What clustering is — and isn't.

Core Idea

Clustering is the art of finding groups in unlabeled data. You have no target variable, no correct answer, no teacher. You just have points in some feature space, and you want to discover which ones belong together based on how close or similar they are.

This is unsupervised learning in its purest form. A classifier tells you *what class a point is*; a clusterer tells you *which other points look like this one*. The distinction matters because clustering has no "accuracy" — there's nothing to be right about. You can only evaluate whether the groups are internally coherent and externally well-separated.

Approach 01
Centroid-based
Cluster = a representative point (centroid) plus whatever is closest to it. K-means is the canonical example. Fast, simple, assumes spherical clusters of similar size.
Approach 02
Hierarchical
Build a tree (dendrogram) of nested clusters by repeatedly merging (agglomerative) or splitting (divisive). HAC is the agglomerative version. No K required up front.
Approach 03
Density-based
Cluster = a dense region of points separated from other dense regions by sparser space. DBSCAN is the classic. Finds arbitrarily shaped clusters, handles noise, doesn't need K.
Clustering has no ground truth. Choosing an algorithm is choosing what the word "similar" means.
Question 01 of 10 — True / False
True or False: Clustering is a supervised learning task because it produces a label (a cluster assignment) for each data point.
Correct — False. The "label" a clusterer produces is not a ground-truth target — it's an assignment the algorithm invented from the data itself. Supervised learning requires labeled training data with known targets. Clustering has no such targets; it finds structure in unlabeled data. Cluster IDs are arbitrary and don't correspond to any meaningful category unless you assign meaning after the fact.
Chapter Two

K-means — the workhorse.

The Algorithm

K-means partitions data into K clusters by repeatedly alternating between two steps until nothing changes. The algorithm is almost embarrassingly simple, which is part of why it dominates.

1. Initialize: randomly choose K initial centroids (or use K-means++ for smarter seeds).
2. Assign: assign each point to the nearest centroid (usually by Euclidean distance).
3. Update: move each centroid to the mean of the points assigned to it.
4. Repeat steps 2–3 until assignments stop changing (convergence).

The objective K-means minimizes is the within-cluster sum of squared distances — also known as inertia or WCSS. Every iteration is guaranteed to decrease inertia or leave it unchanged, so the algorithm always converges — but not necessarily to the global optimum.

The limitations you must memorize.

Limit 01
K must be chosen a priori
You have to pick K before you start. Methods like the elbow method (plot inertia vs K, look for the bend) or silhouette analysis help, but there's no free lunch.
Limit 02
Assumes spherical clusters
K-means implicitly uses Euclidean distance to a centroid, which carves space into Voronoi cells. It fails spectacularly on elongated, nested, or oddly shaped clusters.
Limit 03
Sensitive to initialization
Different starting centroids can lead to different final clusterings. Standard practice: run multiple times with different seeds (n_init) and keep the result with lowest inertia.
Limit 04
Sensitive to outliers & scale
Outliers drag centroids. Unscaled features let large-range features dominate the distance calculation. Standardize first.
Question 02 of 10 — True / False
True or False: The K-means algorithm is guaranteed to find the global minimum of the within-cluster sum of squared distances (inertia).
Correct — False. K-means is a greedy local-search algorithm. It is guaranteed to converge to a local minimum of inertia, but the result depends on the initial centroid placement — different seeds can give different answers. This is why sklearn runs it with multiple random initializations (n_init) by default and keeps the best result. Notice the parallel to decision trees: both are greedy, both find local optima only.
Question 03 of 10
Which distance metric does standard K-means implicitly assume when computing cluster assignments?
Correct — Euclidean (squared) distance. K-means minimizes the sum of squared Euclidean distances from each point to its centroid. The centroid update step (taking the arithmetic mean) is only optimal for squared Euclidean distance — it's not the best summary under Manhattan distance (where the median would be better, giving "K-medians") or under cosine similarity.
Chapter Three

Hierarchical clustering & linkages.

HAC Algorithm

Hierarchical Agglomerative Clustering (HAC) builds clusters bottom-up. It starts with every data point as its own cluster and then repeatedly merges the two closest clusters until only one cluster remains. The history of merges is captured in a dendrogram — a tree where the height of each merge reflects how far apart the merged clusters were.

1. Place every point in its own singleton cluster.
2. Find the two closest clusters using your chosen linkage.
3. Merge them.
4. Repeat until one cluster remains.
5. Cut the dendrogram at any height to get that many clusters.

The critical choice is how you define "closest" between two clusters — the linkage criterion. Different linkages produce genuinely different clusterings from the same data.

The four linkages to know cold.

Linkage Definition Produces clusters that are… Watch out for
Single Distance between the two closest points, one from each cluster (min) Long, stringy, chain-like Chaining: one bridge point can link two otherwise separate clusters
Complete Distance between the two farthest points (max) Compact, tight, spherical Breaks large clusters prematurely; sensitive to outliers
Average Mean pairwise distance between all cross-cluster point pairs Balanced compromise between single and complete More computationally expensive
Ward Merge that causes the smallest increase in total within-cluster variance Compact, similar-sized, spherical (like K-means) Requires Euclidean distance; can't use arbitrary metrics

Question 04 of 10
Which linkage criterion is most prone to the "chaining" effect, where two otherwise distinct clusters get merged because a few intermediate points bridge them?
Correct — single linkage. Because single linkage uses the minimum distance between any two points across clusters, a single close pair is enough to trigger a merge. Put differently: if you can walk from cluster A to cluster B via a chain of short hops, single linkage treats them as one cluster. Complete and Ward avoid this by requiring all pairs (or variance) to be small.
Question 05 of 10 — True / False
True or False: Hierarchical Agglomerative Clustering requires you to specify the number of clusters K before running the algorithm.
Correct — False. This is one of HAC's biggest practical advantages over K-means. HAC produces the entire dendrogram regardless of K. You pick K after the fact by cutting the dendrogram at a chosen height — or you don't pick K at all and keep the hierarchy itself as the output. This lets you see structure at multiple scales with a single run.
Chapter Four

DBSCAN — density over distance.

Density-Based

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) takes a fundamentally different philosophy: a cluster is a region where points are densely packed, surrounded by less-dense space. It does not assume any shape, does not require K, and explicitly identifies outliers as noise.

DBSCAN is controlled by two hyperparameters:

ε (epsilon): the radius defining a point's neighborhood.
minPts: the minimum number of points required in an ε-neighborhood for a region to count as "dense."

The three types of points.

Type 01
Core Point
A point with at least minPts neighbors within its ε-radius (counting itself). These are the "interior" of clusters — densely surrounded.
Type 02
Border Point
A point that is within ε of a core point but does not itself have enough neighbors to be core. It lives on the "edge" of a cluster.
Type 03
Noise (Outlier)
A point that is neither core nor border. Too isolated to belong to any cluster. DBSCAN assigns it cluster ID −1. This is the only algorithm we cover that explicitly flags outliers.

Question 06 of 10
In DBSCAN, a point P has 3 other points within distance ε of itself (4 total including itself). If minPts = 5, and P is within ε of a core point Q, what kind of point is P?
Correct — Border point. P's neighborhood has only 4 points total, below the minPts threshold of 5 — so P is not core. But P is within ε of Q, a core point, which is exactly the definition of a border point. If P were neither core nor within ε of any core point, it would be noise.
Question 07 of 10 — True / False
True or False: DBSCAN requires the user to specify the number of clusters K before running.
Correct — False. This is DBSCAN's signature feature. The number of clusters emerges from the data and the (ε, minPts) settings — you don't specify it. DBSCAN instead asks: "what counts as dense?" Its hyperparameters control density, not cluster count. (Contrast with K-means, which demands K upfront.)
Chapter Five

Evaluation metrics.

How Good Is This Clustering?

Without ground-truth labels, you can't compute "accuracy." Instead you measure intrinsic quality — how internally coherent and externally well-separated the clusters are, using only the data and the cluster assignments.

Inertia (within-cluster sum of squares).

Also: WCSS, Distortion

Inertia is the sum of squared distances from each point to its assigned centroid:

Inertia = Σᵢ ||xᵢ − μ_{c(i)}||²

Lower is better — it measures how tight clusters are around their centers. But note: inertia always decreases as K increases. With K = N you get zero inertia (every point is its own centroid). So you can't pick K by minimizing inertia directly — you use the elbow method: plot inertia vs K and look for where the curve bends.

Silhouette score.

Bounded [-1, 1]

The silhouette score compares how close each point is to its own cluster versus the nearest other cluster. For a point i:

a(i) = mean distance to other points in i's own cluster
b(i) = mean distance to points in the nearest other cluster
s(i) = (b(i) − a(i)) / max(a(i), b(i))

The overall silhouette score is the mean of s(i) over all points.

s ≈ 1
Well-clustered
Point is much closer to its own cluster than to any other. This is what you want.
s ≈ 0
On the boundary
Point sits roughly halfway between two clusters. Could plausibly go either way.
s ≈ −1
Likely misassigned
Point is closer to another cluster than to its own. It probably belongs there.

Question 08 of 10 — Calculation
For a point in a clustering, you compute a(i) = 2.0 (mean distance to its own cluster) and b(i) = 5.0 (mean distance to the nearest other cluster). What is the silhouette value s(i)?
Correct — 0.6. Using s(i) = (b − a) / max(a, b): s(i) = (5.0 − 2.0) / max(2.0, 5.0) = 3.0 / 5.0 = 0.6 Positive and reasonably high — the point sits much closer to its own cluster than to the next one. A value of 0.4 would result from accidentally dividing by (a + b) instead of max. A negative value would mean the point is closer to another cluster than its own.
Question 09 of 10 — True / False
True or False: Since lower inertia means tighter clusters, the best K is always the one with the lowest inertia.
Correct — False. Inertia always decreases as K increases — with K = N (every point its own cluster), inertia hits zero. So blindly minimizing inertia picks K = N, which is useless. That's why we use the elbow method: look for the K where the curve stops dropping sharply and starts leveling off. This is a classic Snell-style gotcha: something that sounds right ("lower inertia = better clustering") has a fatal subtlety.

Chapter Eight

Final self-check.

Question 10 of 10 — Synthesis
You have a 2D dataset shaped like two interleaving crescent moons, plus about 5% obvious outliers scattered throughout. You want to recover exactly two clusters and label the outliers as noise. Which algorithm is most appropriate?
Correct — DBSCAN. Two reasons: (1) the crescent shapes are non-convex and non-spherical, so K-means and Ward (which prefer spherical clusters) will cut across both moons rather than separating them. Complete linkage will also struggle with elongated shapes. (2) Only DBSCAN can label outliers as noise — the other three algorithms assign every point to a cluster. Single-linkage HAC could handle the crescent shapes but would still force the outliers into clusters and is vulnerable to chaining. DBSCAN is the tool built for exactly this situation.
