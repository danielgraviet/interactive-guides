§ 01

The Curse of Dimensionality

As dimensions grow, geometry stops behaving the way your intuition expects. Data becomes sparse, distances become meaningless, and every model you trust starts to fail in strange, quiet ways.

The core idea

The "curse" refers to a bundle of counterintuitive phenomena that emerge as the number of features d grows. The phrase was coined by Richard Bellman in the 1960s, and it shows up everywhere: in nearest-neighbor classifiers, density estimation, optimization, and clustering.

The three faces of the curse

1. Exponential data requirements. To cover a unit hypercube with a fixed density of samples, you need an exponential number of points in d. If 10 points cover a 1D interval, you need 10d points to cover a d-dimensional cube at the same density. A dataset that felt rich in 2D is catastrophically sparse in 50D.

2. Distance concentration. In high dimensions, the distance from a query point to its nearest neighbor and its farthest neighbor become nearly equal. The ratio (max − min) / min → 0. This is why k-NN, k-means, and any method built on Euclidean distance can silently break.

3. Volume concentrates in the shell. In a high-dimensional ball, almost all the volume sits near the surface, not the interior. A thin shell of width ε around a unit ball contains nearly 100% of the volume once d is large. "Typical" points are all near the boundary.

Interactive — Distance Concentration
Common trap

Students often say "the curse of dimensionality means models overfit in high dimensions." That's a consequence, not the definition. The curse is fundamentally geometric — about how distance, volume, and density behave. Overfitting is one downstream symptom.

Key insight

High-dimensional data rarely fills its ambient space. Real data usually lies on a much lower-dimensional manifold. This is the "blessing" that makes dimensionality reduction possible at all: the intrinsic dimension is smaller than d.

Question · Conceptual
In a dataset with 1,000 features, a student finds that their k-NN classifier performs no better than random guessing. Which explanation is most precise?
C is correct. This is the distance-concentration face of the curse. For k-NN, the ratio of the farthest to nearest distance approaches 1 in high-D, so the notion of "closest point" loses discriminative power. A is a different concept (model complexity), B is wrong (k-NN is perfectly good in low-D), and D could be true for any algorithm and isn't specific to the symptom described.
Question · Geometric
Consider a 100-dimensional unit ball. Roughly what fraction of its volume lies within the outer shell of thickness 0.1 (i.e., radius between 0.9 and 1.0)?
C is correct. The volume of a d-ball scales with rd. The inner ball of radius 0.9 holds a fraction 0.9100 ≈ 0.0000266 of the total volume. So >99.99% of the volume sits in the outer 10% shell. This is why "typical" points in high-D are nearly all at the boundary — a huge violation of 2D/3D intuition.
§ 02

Feature Selection

Choose a subset of the original features. Throw the rest away. The features that survive still mean exactly what they meant before.

Why selection over reduction?

Feature selection preserves interpretability. If your dataset has columns like "age," "blood pressure," and "cholesterol," the features you keep are still literally age, blood pressure, and cholesterol. Your doctor can read the model. Contrast this with PCA, where you end up with "0.4 × age − 0.7 × BP + 0.6 × chol" — mathematically fine, clinically opaque.

Three families of methods

FamilyHow it decidesExampleTradeoff
Filter Score each feature independently using a statistic (correlation, mutual info, chi-squared). Ignores the model. Rank features by Pearson correlation with target, keep top k. Fast, model-agnostic. Misses feature interactions — two weak features can be strong together and filters will drop both.
Wrapper Train the model on candidate subsets and use its performance as the score. Treats the model as a black box. Forward selection, backward elimination, recursive feature elimination (RFE). Catches interactions, finds subsets tuned to the model. Expensive — exponential in the worst case, risk of overfitting the selection itself.
Embedded Selection happens during model training as part of the learning objective. L1 (Lasso) regularization, tree-based feature importance. Best of both worlds in practice — integrated with training, usually efficient. Tied to a specific model family.
Common trap

A feature with zero correlation to the target is not necessarily useless. Filter methods will drop it, but it might be essential in combination with another feature (classic example: XOR — each input is independently uncorrelated with the output but together determine it perfectly). This is the fundamental blind spot of univariate filters.

Question · Method classification
You fit a Lasso (L1-regularized) linear regression and observe that 40 of your 100 features have coefficients of exactly zero. This is an example of which type of feature selection?
C is correct. L1 regularization is the textbook example of an embedded method — selection is baked into the loss function itself, happening during training rather than before (filter) or around (wrapper) it. And importantly, it is selection, not reduction: the surviving features are still the original features with their original meanings. D is the tempting wrong answer if you've confused "shrinks the feature set" with "transforms the feature space."

Question · Why it fails
A filter method ranks features by their individual correlation with the target and keeps the top 10. Which scenario is this approach most likely to handle poorly?
B is correct. Filter methods evaluate features one at a time. In an XOR relationship, each input feature has near-zero individual correlation with the target, so the filter will discard both — destroying the signal. This is the canonical failure mode and exactly what wrapper and embedded methods are designed to catch.
§ 03

Feature Reduction

Rather than picking a subset, construct new features as combinations of the old ones. Fewer dimensions, but each dimension is a hybrid.

Selection vs. Reduction — the crucial distinction

This distinction is a classic MCQ trap. Both shrink the feature count, but they do it differently:

Feature SelectionFeature Reduction (Extraction)
What it outputsA subset of original featuresNew combinations (usually linear or nonlinear projections)
InterpretabilityHigh — features keep their meaningsLower — new features are mixtures
Information lossDiscards entire columnsCan preserve most of the variance across all original columns
Typical methodsLasso, RFE, mutual info rankingPCA, t-SNE, UMAP, autoencoders, LDA
When to useWhen interpretability matters or features are cheap to computeWhen you need maximal compression or are feeding into a downstream model

Linear vs. nonlinear reduction

Reduction methods also split along the linear / nonlinear axis. PCA and LDA are linear — the new features are weighted sums of the old. t-SNE, UMAP, and autoencoders are nonlinear — they can capture curved manifolds that linear projections would flatten into uselessness.

Key insight

Feature reduction is fundamentally about finding a lower-dimensional representation that preserves some structure. What structure? It depends on the method:

PCA preserves variance. LDA preserves class separability. t-SNE / UMAP preserve local neighborhoods. Each answers a different question, and choosing the wrong one for your task gives misleading results.

Question · Definitional
Which of the following is a feature reduction technique, not a feature selection technique?
C is correct. PCA creates new features (principal components) that are linear combinations of the originals — this is reduction/extraction. A, B, and D all retain or discard original features unchanged, which makes them selection.
§ 04

Variance and Covariance

Before PCA makes sense, you need to feel what variance and covariance are — not just compute them, but see the ellipse they draw.

Variance — how far a single variable spreads

For a single feature X, variance measures the average squared deviation from the mean:

Var(X) = E[(X − μ)²] = (1/n) Σ (xᵢ − x̄)²

Variance is always ≥ 0. It is measured in the squared units of X. Standard deviation (its square root) lives in the original units and is often more interpretable.

Covariance — how two variables move together

Cov(X, Y) = E[(X − μX)(Y − μY)]

The sign tells you the relationship type. The magnitude depends on the scales of X and Y, which makes raw covariance hard to compare across variable pairs.

  • Positive: when X is above its mean, Y tends to be above its mean.
  • Negative: when X is above its mean, Y tends to be below its mean.
  • Zero (or near zero): no linear relationship — but nonlinear relationships can still exist.

The covariance matrix

For a dataset with d features, the covariance matrix Σ is a d × d symmetric matrix where:

  • Diagonal entries Σii = Var(Xi) — the variance of each feature.
  • Off-diagonal entries Σij = Cov(Xi, Xj) — pairwise covariances.
  • The matrix is always symmetric and positive semi-definite.
Interactive — covariance shapes the scatter
Common trap

Zero covariance ≠ independence. Covariance detects only linear relationships. Two variables can be perfectly deterministically related (e.g., Y = X² with X symmetric around zero) and still have covariance exactly zero. Independence implies zero covariance; the reverse is not true (except for jointly Gaussian variables, where they coincide).

Common trap

Covariance is scale-dependent. If you change X from meters to millimeters, its covariance with Y grows by 1000×. For scale-free comparison, use the Pearson correlation ρ = Cov(X,Y) / (σX σY), which always lies in [−1, 1]. This is also why PCA requires standardization when features have different units.

Question · Conceptual
Two random variables X and Y satisfy Cov(X, Y) = 0. Which statement is guaranteed?
B is correct. Covariance captures linear association only. A is the classic trap — zero covariance does not imply independence (except in the jointly Gaussian case). A deterministic nonlinear relationship like Y = X² with symmetric X has zero covariance but full dependence.
Question · Matrix structure
Which of the following is never true of a valid covariance matrix?
C is correct. A covariance matrix is positive semi-definite, meaning all eigenvalues are ≥ 0. A negative eigenvalue would correspond to some linear combination of variables having negative variance — impossible. A, B, and D are all legal (off-diagonal entries can absolutely be negative — that just means the pair of variables has negative covariance).
§ 05

Principal Component Analysis

Find the directions your data stretches farthest. Call those axes. Forget the rest.

The intuition in one sentence

PCA rotates the coordinate system so that the new axes — the principal components — point along directions of maximum variance. The first PC captures the most variance, the second captures the most remaining variance while being orthogonal to the first, and so on.

The recipe

  1. Center the data — subtract the mean of each feature. (Critical. Skipping this gives you the wrong answer.)
  2. Standardize if features are on different scales (divide by each feature's std. dev.).
  3. Compute the covariance matrix Σ of the centered data.
  4. Find its eigenvectors and eigenvalues. Sort eigenvalues in descending order.
  5. The top k eigenvectors form the projection matrix W.
  6. Project: Xnew = X W. Done.

Equivalently, you can skip explicitly forming the covariance matrix and apply SVD directly to the centered data matrix: X = UΣVᵀ, and the columns of V are your principal components. This is what libraries use in practice — it's numerically more stable.

The variance / eigenvalue connection

This is the heart of PCA and a favorite exam target:

The eigenvalue λi of the covariance matrix equals the variance of the data along the corresponding principal component.

So the "explained variance ratio" of the i-th component is λi / (Σ λj). A scree plot shows these ratios and you typically pick k at the elbow or at a cumulative threshold (e.g., 95%).

What PCA is — and isn't

PCA ISPCA IS NOT
A linear transformation Nonlinear — it cannot unfold a manifold like a Swiss roll
Unsupervised — uses no label information Class-aware — that's LDA, which maximizes class separability
Variance-maximizing Distance-preserving in general (except in the subspace it keeps)
Sensitive to feature scale — so standardize first A feature selector — it transforms all features into new ones
Invertible up to reconstruction error Lossless (unless you keep all d components)
Common trap

PCA ≠ best for classification. The direction of maximum variance is not necessarily the direction that separates classes. A common exam setup: imagine two class clusters stretched along one axis but separated along another. PCA will keep the stretch direction (high variance) and throw away the separation direction (lower variance). LDA is the supervised alternative that uses class labels to find the separating direction.

Common trap

Forgetting to center the data gives mathematically valid eigenvectors of XᵀX, but they no longer correspond to directions of variance — they correspond to directions of largest overall magnitude, which is a different thing. Forgetting to standardize lets a single high-variance feature (e.g., income in dollars vs. age in years) dominate PC1 for purely unit-based reasons.

Question · Core mechanics
In PCA, the principal components are the eigenvectors of the covariance matrix. The corresponding eigenvalues represent:
B is correct. Each eigenvalue equals the variance of the projected data along its eigenvector. This is why we sort eigenvalues in descending order and keep the top k — we keep the k directions with highest variance. C is wrong because PCA is unsupervised and never sees the target label.
Question · Failure mode
Consider a binary classification problem where classes are arranged like this: within-class spread is 10 along direction A, and the two class centers are separated by 2 along direction B (orthogonal to A). You apply PCA and keep only the first principal component. What happens?
B is correct. PCA is unsupervised and maximizes variance. Direction A has much higher variance (spread of 10) than direction B (spread of 2), so PC1 aligns with A — the nonseparating direction. Projecting onto PC1 collapses the class-separation information entirely. This is the canonical "PCA for classification can fail" example and a textbook motivator for LDA.
Question · Preprocessing
You have a dataset with features: "age" (years, range 20–70), "income" (dollars, range 30,000–200,000), and "height" (cm, range 150–200). You run PCA without standardizing first. What is the most likely outcome?
B is correct. Variance is computed in the feature's own units, so a feature measured in dollars will have vastly larger raw variance than one in years or centimeters. PC1 will collapse onto income almost entirely — not because income is "important," but because of its units. This is why standardization (z-score) is essentially mandatory when features have different scales.
§ 06

t-Distributed Stochastic Neighbor Embedding

A nonlinear method obsessed with one thing: preserving who your neighbors are. It makes beautiful cluster pictures. It also lies to you in very specific ways, and you need to know which ones.

The intuition

t-SNE (2008, van der Maaten & Hinton) tries to place points in low-D such that points that were close in high-D stay close, and points that were far in high-D stay far. It does this probabilistically: it turns pairwise distances into probabilities of "being neighbors," and matches the high-D neighbor distribution to the low-D neighbor distribution.

The mechanism, in pieces

1. High-D neighbor probabilities. For each point, place a Gaussian around it in the high-D space. The probability pj|i that point j is a neighbor of point i is proportional to the Gaussian density at j. The Gaussian's bandwidth is tuned per-point via a parameter called perplexity, which roughly equals "effective number of neighbors."

2. Low-D neighbor probabilities. In the low-D space, do the same but with a Student's t-distribution (with 1 degree of freedom = Cauchy). The t-distribution has heavier tails than a Gaussian, which gives far-apart points extra "room" to be placed far apart without penalty — this is what prevents the infamous "crowding problem" that afflicted the earlier SNE algorithm.

3. Match the distributions. Minimize the KL divergence between the high-D and low-D neighbor distributions using gradient descent. Points iteratively move in the low-D space until neighborhoods align.

Interactive — the effect of perplexity

What t-SNE preserves — and what it doesn't

Critical exam territory

t-SNE preserves local structure. It does not preserve global structure.

This has specific, testable consequences:

  • Cluster sizes are meaningless. A tiny cluster in the t-SNE plot may not be a "denser" cluster in the original data — t-SNE expands and contracts regions to suit its neighbor-matching objective.
  • Distances between clusters are meaningless. Two clusters appearing far apart in the plot may be close in the original space, or vice versa.
  • Perplexity dramatically changes the picture. Different perplexities can produce qualitatively different plots of the same data. It is not a method of absolute truth.
  • It's stochastic. Different random seeds produce different embeddings. Run it more than once.
Another trap

t-SNE does not learn a mapping function. If you add a new data point later, you cannot simply apply t-SNE to project it — you need to re-run the whole optimization (or use an approximation like parametric t-SNE). This is a fundamental difference from PCA, which gives you an explicit projection matrix you can reuse.

Question · What it preserves
You produce a 2D t-SNE plot. Three clusters appear: A, B, and C. In the plot, A and B are close to each other, and C is far across the canvas. What can you conclude about the original high-dimensional relationships?
C is correct. t-SNE is tuned to preserve local neighborhoods, not global geometry. The distance between two clusters on a t-SNE plot is essentially an artifact of the optimization and has no reliable relationship to their high-D distance. A and D are exactly the kind of overinterpretation the van der Maaten / Wattenberg "How to Use t-SNE Effectively" paper warns against.
Question · Mechanism
Why does t-SNE use a Student's t-distribution in the low-dimensional space instead of a Gaussian?
B is correct. The heavy tails are the whole point. When you compress high-D space into 2D, there isn't enough "room" for all the distant points — in a Gaussian scheme, moderate distances get attractive forces that crowd everything together. The t-distribution (specifically Cauchy, df=1) assigns higher probability to large distances, meaning distant points can stay distant in 2D without incurring a huge penalty. This is the crowding problem solution and the key innovation over the original SNE.
Question · Practical usage
You train a model and want to deploy it. As new data streams in, you want to visualize where each new point lands relative to your training data. What's the problem with using vanilla t-SNE for this?
B is correct. Vanilla t-SNE doesn't learn a function f: high-D → low-D; it just positions a specific set of points. New points don't have coordinates, and you'd have to re-optimize (and even then, the previous points could shift). Contrast with PCA (explicit projection matrix) and UMAP (can fit + transform new data).
§ 07

Uniform Manifold Approximation and Projection

The spiritual successor to t-SNE — faster, scalable to millions of points, preserves more global structure, and built on a pile of topology that you do not need to fully understand to use well.

What UMAP brings to the table

UMAP (2018, McInnes et al.) was designed to address t-SNE's main limitations. It builds a graph of the data's local neighborhoods in high-D, then finds a low-D layout whose graph structure matches. The key advantages:

  • Much faster than t-SNE on large datasets — can embed millions of points.
  • Preserves more global structure — between-cluster distances in UMAP plots are more meaningful than in t-SNE (though still not fully trustworthy).
  • Produces a reusable mapping. You can fit UMAP on training data and then transform new data points into the same embedding space — something t-SNE cannot do.
  • Flexible output dimension. Useful not just for 2D/3D visualization but also as a general-purpose dimensionality reduction tool feeding into downstream models.

The two main parameters

ParameterWhat it controlsEffect
n_neighbors Size of the local neighborhood used to construct the graph. Small (5–15): focuses on very local structure, fragments the data into many small pieces. Large (50+): captures more global structure, may smear out fine detail.
min_dist Minimum distance points are allowed to occupy in the low-D embedding. Small (0.0–0.1): tightly packs similar points, emphasizes cluster structure. Large (0.5+): spreads points out, emphasizes broad topological relationships.

UMAP vs. t-SNE — the comparison you'll be tested on

t-SNEUMAP

Key insight

Both t-SNE and UMAP are manifold learning methods — they assume the data lies on a curved surface embedded in high-D space, and they try to flatten that surface into 2D or 3D. PCA, by contrast, is strictly linear and would rip apart a curved manifold like the classic Swiss roll.

Common trap

UMAP is still imperfect on global structure. It's better than t-SNE, but cluster-to-cluster distances are still not a reliable readout of high-D distances. Students sometimes read "UMAP preserves global structure" as "UMAP plots can be interpreted quantitatively." They can't be. Treat all UMAP/t-SNE plots as qualitative, not quantitative.

Question · Method comparison
Which of the following is a capability of UMAP that vanilla t-SNE lacks?
C is correct. UMAP's fit/transform paradigm is a major practical advantage — you can fit on training data and then map new test points onto the same embedding. Vanilla t-SNE has to re-optimize from scratch. D is a distractor: UMAP also has stochasticity and produces different results with different seeds.
Question · Tuning
A researcher runs UMAP with n_neighbors=5 and gets many small, fragmented clusters. They want the visualization to reflect more of the broad, global structure of the dataset. What should they do?
B is correct. n_neighbors controls the local/global tradeoff. Small values zoom in on tight local structure and fragment the dataset; large values widen the neighborhood and emphasize broad topology. C would actually tighten clusters further (concentrates points). D is not clearly true — PCA preserves variance, which is a different notion of "global."
Question · Mixed — final boss of this section
A student applies both PCA and UMAP to the same dataset of 50,000 points with 200 features. They find that UMAP reveals three distinct clusters, but PCA with 2 components does not separate them at all. Which interpretation is most accurate?
C is correct. PCA is strictly linear — if clusters are separated along a curved manifold (think Swiss roll, or concentric rings), a linear projection will mix them together while UMAP can unfold the manifold. This scenario is a textbook motivator for nonlinear methods. B is too strong — UMAP can hallucinate structure with bad parameters, but when the result is stable across parameters and seeds, it's usually real. A is overconfident in the opposite direction.

Mixed Final Exam

All seven topics, unpredictable order, harder traps. This is what the professor's MCQs look like.

Question 1 · Mixed
Which of the following statements is true?
C is correct. A is wrong (PCA is reduction/extraction, not selection). B is wrong (both have stochastic components). D is wrong (Lasso is embedded, not wrapper).
Question 2 · Mixed
You want to reduce 1,000 features to 2 dimensions for a scatter plot, and you need the plot to reflect the global geometry of the data (cluster-to-cluster distances should be meaningful). Which method is the best fit?
A is correct. Of the reduction methods, PCA best preserves global linear structure (variance directions), and distances in its projection are metric. t-SNE explicitly destroys global structure. UMAP is better than t-SNE globally but still not metric. C and D are selection methods and don't create new axes.
Question 3 · Mixed
A dataset has 5 features. Three of them are nearly identical copies of each other (high pairwise correlation). You apply PCA. Roughly what should happen?
B is correct. Correlated features share variance, and PCA is designed to collapse that shared variance into a single direction. The three near-copies will largely project onto one principal component. C is backwards — PCA loves correlated features; that's when reduction is most effective. D confuses reduction with selection (PCA doesn't drop features, it combines them).
Question 4 · Mixed
Your colleague claims: "I ran t-SNE with perplexity 30, and cluster A has diameter 5 while cluster B has diameter 15, so cluster B has three times the spread of cluster A in the original data." What's wrong?
B is correct. t-SNE preserves local neighborhoods, not distances or sizes. A dense cluster will be drawn differently than a sparse one, and the relative sizes of clusters on the plot don't correspond to their original spreads. This is one of the most frequently cited misuses of t-SNE.
Question 5 · Mixed
Which statement correctly distinguishes feature selection from feature reduction?
C is correct. B is exactly reversed. A is false (PCA is very fast). D is false (both families contain supervised and unsupervised methods; e.g., mutual-info selection is supervised, PCA reduction is unsupervised).
Question 6 · Mixed
A covariance matrix with 100 features has eigenvalues sorted in descending order: λ₁ = 50, λ₂ = 30, λ₃ = 10, λ₄ = 5, λ₅ through λ₁₀₀ ≈ 0. You want to retain 90% of the total variance via PCA. How many components do you keep?
B is correct. Total variance = sum of all eigenvalues ≈ 50 + 30 + 10 + 5 = 95 (others are negligible). Cumulative explained variance: PC1 = 50/95 ≈ 52.6%, PC1+2 = 80/95 ≈ 84.2%, PC1+2+3 = 90/95 ≈ 94.7%. So 3 components gets you past 90%.
Question 7 · Mixed
You train an image classifier on 50×50 grayscale images (2,500 features). You need to visualize the data in 2D to inspect cluster structure and you suspect classes lie on a nonlinear manifold. You also need to project new test images into the same space for live monitoring. Best choice?
C is correct. Three constraints: (1) 2D visualization — all options can do this. (2) Nonlinear manifold — rules out PCA (linear) and Lasso (selects pixels, not combinations). (3) Must project new test data — rules out t-SNE, which has no reusable mapping. Only UMAP satisfies all three.
Question 8 · Mixed
You have data with 3 features. Cov(X₁,X₂) = 0, Cov(X₁,X₃) = 0, Cov(X₂,X₃) = 0, and all three have equal variance. Running PCA on this dataset would:
B is correct. The covariance matrix is a scalar multiple of the identity (equal diagonal, zero off-diagonal). Its eigenvalues are all equal, and every direction has the same variance — there is no "preferred" direction. PCA gives you a valid orthogonal basis but no reduction opportunity: dropping any component costs exactly 1/3 of the variance. This is PCA's fundamental limitation: it requires redundancy (correlation) in the features to be useful.
