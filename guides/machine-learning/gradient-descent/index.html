01 — Core Concepts

How Gradient Descent Works

⛰️

The Hill Analogy

Imagine you're blindfolded on a hilly landscape and want to reach the lowest valley. You feel the slope under your feet (the gradient) and take a step downhill. Gradient descent does exactly this — repeatedly nudging weights in the direction that reduces loss.

1

Forward Pass — Compute Loss

Run your input through the network, get a prediction, then compute how wrong you were using a loss function (e.g., MSE, Cross-Entropy).

2

Backward Pass — Compute Gradient

Use backpropagation (chain rule) to calculate the partial derivative of the loss with respect to each weight: ∂L/∂w. This is the gradient — the slope of the loss landscape at your current position.

3

Update Weights

Subtract the gradient (scaled by learning rate α) from each weight. If the gradient is negative (slope goes down to the right), increase the weight.

4

Repeat Until Convergence

Keep iterating until the loss stops meaningfully decreasing — you've reached (a local) minimum.

The Update Rule
w_new = w_old − α · (∂L/∂w)

w = weight  |  α = learning rate  |  ∂L/∂w = gradient of loss w.r.t. weight
💡

The Sign Intuition (Professor loves this!)

Negative gradient → loss curve goes downhill to the right → increase the weight (subtract a negative = add).
Positive gradient → loss curve goes uphill to the right → decrease the weight (subtract a positive).
You always move opposite to the gradient — that's why it's called gradient descent, not ascent.

02 — Key Formulas

Derivatives You Need to Know

📉 Loss Functions

Mean Squared Error (MSE)
L = (1/n) Σ (ŷ − y)²
∂L/∂ŷ = (2/n)(ŷ − y)
Binary Cross-Entropy
L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]
∂L/∂ŷ = (ŷ − y) / [ŷ(1−ŷ)]
Categorical Cross-Entropy
L = −Σ yᵢ · log(ŷᵢ)
∂L/∂ŷᵢ = −yᵢ/ŷᵢ

⚡ Activation Gradients

Sigmoid σ(z) = 1/(1+e⁻ᶻ)
σ'(z) = σ(z) · (1 − σ(z))
tanh
tanh'(z) = 1 − tanh²(z)
ReLU max(0, z)
ReLU'(z) = 1 if z > 0, else 0
Leaky ReLU
f'(z) = 1 if z > 0, else α (small const)

🎯 Worked Example — Weight Update

Given: w = 0.5, α = 0.01, gradient ∂L/∂w = −2.3
w_new = 0.5 − 0.01 × (−2.3) = 0.5 + 0.023 = 0.523

The gradient is negative → slope goes downhill to the right → we increase w to move toward the minimum. ✓

03 — Types of Gradient Descent

Batch vs SGD vs Mini-Batch

The Study Analogy

Batch GD — Solve all 1,000 practice problems, then reflect on every mistake at once before adjusting strategy. Very thorough but slow.
SGD — Solve one problem, immediately adjust, then move to the next. Fast but one weird problem can throw you off.
Mini-Batch — Solve 32 problems, reflect on those, adjust, repeat. The sweet spot in practice.

Property Batch GD Stochastic GD Mini-Batch GD
Data per update All N samples 1 sample k samples (e.g., 32, 64)
Gradient accuracy High Noisy Medium
Computation/update Slow Fastest Fast
Iterations/epoch 1 N (many!) N/k
Outlier sensitivity Low High Medium
Memory usage High Lowest Low
Overfitting risk Higher Lower Medium
Used in practice? Rarely Online learning Most common

SGD Properties

  • Updates weights one instance at a time
  • Sensitive to outliers in the data
  • Fastest computation per iteration
  • Avoids overfitting (noisy updates act like regularization)
  • Takes many iterations per epoch
  • Path to minimum is zig-zaggy

Mini-Batch Properties

  • Typical batch sizes: 32, 64, 128, 256
  • Works well with GPU vectorization
  • Smoother convergence than SGD
  • Batch size is a hyperparameter
  • Default in most deep learning frameworks

Batch GD Properties

  • Uses entire dataset per update
  • Deterministic — same result every run
  • Guaranteed smooth loss curve
  • Impractical for large datasets
  • Not the default in practice
04 — Problems in Gradient Descent

Vanishing & Exploding Gradients

💨 Vanishing Gradient

Occurs when gradients become extremely small as they propagate backward through the network.

Why it happens
grad = δ₁ × δ₂ × δ₃ × ... × δₙ
If each δᵢ = 0.1 → 0.1⁵ = 0.00001 ≈ 0

Cause: Lots of small numbers (like sigmoid gradients max at 0.25) multiplied together approach 0. Early layers barely learn.

Deep networks Sigmoid/tanh Early layers starved

💥 Exploding Gradient

Occurs when gradients become extremely large, causing massive unstable weight updates.

Why it happens
grad = δ₁ × δ₂ × δ₃ × ... × δₙ
If each δᵢ = 5 → 5⁵ = 3125 → NaN

Cause: Large weight values combined with many layers cause gradients to grow exponentially. Poor initial weights are a key trigger.

Poor initialization RNNs Loss → NaN

🔑 Why Poor Initial Weights → Exploding Gradients

If weights are initialized too large, the activations in each layer will also be large. During backpropagation, gradients are multiplied by these large weight values at each layer (via the chain rule). With many layers, each multiplication amplifies the gradient further, leading to exponential blowup. Networks with poor initial weights are therefore especially susceptible to exploding gradients.

🛠 Solutions

Against Vanishing

  • Use ReLU instead of sigmoid/tanh
  • Batch Normalization — rescales activations
  • Residual connections (skip connections)
  • Careful weight initialization (He, Xavier)

Against Exploding

  • Gradient Clipping — cap gradients at a max value
  • Proper weight initialization
  • Smaller learning rate
  • Batch Normalization helps here too

📦 Batch Normalization Explained

Batch normalization is a technique for training neural networks that rescales the input features at each layer by standardizing them (applying z-score normalization: subtract mean, divide by std deviation). This keeps activations in a healthy range, preventing large gradients and speeding up training. It effectively prevents both vanishing and exploding gradient problems.

Z-Score Standardization (what BatchNorm does)
x̂ = (x − μ_batch) / √(σ²_batch + ε)
output = γ · x̂ + β   ← learnable scale and shift
05 — Variations on Gradient Descent

Beyond Plain GD

Momentum
// adds velocity to avoid local minima
▾

Momentum adds a fraction of the previous update to the current one, like a ball rolling downhill that builds up speed. This helps escape local minima and saddle points, and dampens oscillations.

Update rule:

v = β · v_prev + (1−β) · ∇L
w = w − α · v

β (momentum coefficient) is typically 0.9. A higher β means more "inertia" — the optimizer remembers more of its past direction.

RMSProp
// adaptive learning rates per parameter
▾

RMSProp divides the learning rate by a running average of recent gradient magnitudes. Parameters with large recent gradients get smaller effective learning rates, and vice versa.

E[g²] = β · E[g²]_prev + (1−β) · g²
w = w − α / √(E[g²] + ε) · g

This helps in situations where gradients vary a lot across dimensions (common in non-convex loss surfaces).

Adam (Adaptive Moment Estimation)
// momentum + RMSProp combined — most popular
▾

Adam combines the benefits of both momentum and RMSProp. It maintains a moving average of both the gradient (1st moment) and the squared gradient (2nd moment), with bias corrections.

m = β₁·m + (1−β₁)·g   ← 1st moment (mean)
v = β₂·v + (1−β₂)·g² ← 2nd moment (variance)
m̂ = m/(1−β₁ᵗ)    v̂ = v/(1−β₂ᵗ) ← bias correction
w = w − α · m̂/√(v̂+ε)

Default: β₁=0.9, β₂=0.999, ε=1e-8. Adam is the most commonly used optimizer in deep learning today.

Learning Rate Schedules
// decay α over time for better convergence
▾

Instead of a fixed learning rate, you can decay it over time. Large α early on allows big exploratory steps; smaller α later allows fine-tuned convergence.

Step decay: halve α every N epochs
Exponential decay: α = α₀ · e^(−kt)
Cosine annealing: oscillates α following a cosine curve
Warm restarts: periodically resets α to escape local minima

Nesterov Accelerated Gradient (NAG)
// "look-ahead" momentum
▾

NAG computes the gradient at the anticipated future position (where momentum would take you), not the current position. This "look-ahead" makes updates smarter and convergence faster than standard momentum.

v = β · v_prev + α · ∇L(w − β·v_prev)
w = w − v

Think of it as: regular momentum commits blindly to the direction you're going; NAG first "peeks" ahead and corrects course before committing.

06 — scikit-learn Implementation

GD for Training MLPs in sklearn

MLPClassifier with SGD

Use solver='sgd' for stochastic gradient descent. Control batch size with batch_size.

from sklearn.neural_network import MLPClassifier

# Stochastic Gradient Descent
clf_sgd = MLPClassifier(
    hidden_layer_sizes=(100, 50),
    solver='sgd',
    batch_size=1,        # 1 = true SGD
    learning_rate_init=0.01,
    max_iter=200
)

# Mini-Batch GD
clf_mb = MLPClassifier(
    hidden_layer_sizes=(100, 50),
    solver='sgd',
    batch_size=32,       # 32 = mini-batch
    learning_rate_init=0.01,
    momentum=0.9,         # add momentum
    max_iter=200
)

MLPClassifier with Adam

Use solver='adam' for the Adam optimizer — the sklearn default.

# Adam Optimizer (default)
clf_adam = MLPClassifier(
    hidden_layer_sizes=(100, 50),
    solver='adam',         # default!
    learning_rate_init=0.001,
    beta_1=0.9,
    beta_2=0.999,
    max_iter=200
)

# Batch GD (approx. using lbfgs)
clf_batch = MLPClassifier(
    hidden_layer_sizes=(100, 50),
    solver='lbfgs',        # quasi-Newton
    max_iter=500
)

clf_adam.fit(X_train, y_train)
acc = clf_adam.score(X_test, y_test)

⚠️ sklearn solver options

'sgd' — Stochastic or mini-batch GD. Set batch_size to control.
'adam' — Adam optimizer. Default and recommended for most cases.
'lbfgs' — Quasi-Newton method; good for small datasets, closest to full batch GD.
Note: sklearn uses mini-batch by default when solver='sgd', not truly one-sample-at-a-time SGD.
