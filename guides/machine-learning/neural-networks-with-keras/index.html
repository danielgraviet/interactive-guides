01 // What is Artificial Intelligence?

Definition

Artificial Intelligence is the field of computer science focused on building systems that can perform tasks that typically require human-level intelligence — including reasoning, learning, perception, language understanding, and decision-making. AI systems learn patterns from data and generalize to new situations rather than following rigid explicit rules.

The Six Domain Areas of AI

  • 1. Machine Learning (ML) — Systems that learn from data to improve performance without being explicitly programmed for every case.
  • 2. Natural Language Processing (NLP) — Enabling computers to understand, interpret, and generate human language (text and speech).
  • 3. Computer Vision — Enabling machines to interpret and understand visual information from images and video.
  • 4. Robotics — Designing intelligent agents that sense, reason, and act physically in the real world.
  • 5. Expert Systems / Knowledge Representation — Encoding domain knowledge as rules or graphs so AI can reason symbolically and make inferences.
  • 6. Planning & Search / Decision-Making — Algorithms that find optimal sequences of actions, including game-playing, scheduling, and autonomous agents.

02 // Discriminative vs. Generative Models
Discriminative

Discriminative Models

Learn the boundary between classes. They model P(Y | X) — the probability of a label given input data. They ask: "Given this input, which class does it belong to?"


Examples: Logistic Regression, SVM, Decision Trees, CNNs for classification, BERT (classification head)

Generative

Generative Models

Learn the underlying data distribution. They model P(X, Y) or P(X) — they can generate new samples similar to training data. They ask: "What does this class of data look like?"


Examples: GANs, VAEs, Diffusion Models, GPT, Naive Bayes

03 // Convolutional Neural Network (CNN)

Definition

A CNN is a deep learning architecture designed to process grid-structured data (images, video, audio spectrograms) by exploiting spatial locality and translation invariance. It uses learned filters (kernels) that slide across the input to detect features regardless of their position.

Core Components

  • Convolutional Layer — Applies learned filters to extract local features (edges, textures, shapes). Each filter produces a feature map.
  • Activation (ReLU) — Introduces non-linearity; removes negative values.
  • Pooling Layer — Downsamples feature maps (Max Pool, Avg Pool) to reduce dimensionality and gain spatial invariance.
  • Fully Connected Layer — Final layers that aggregate features for classification or regression output.

Key Properties (Exam-Critical)

  • Parameter sharing: The same filter weights are used across all positions — dramatically fewer parameters than fully connected layers.
  • Local receptive field: Each neuron connects only to a local region of the input.
  • Translation invariance: Can detect features regardless of where they appear in the image.
  • Hierarchical feature learning: Early layers learn low-level features (edges); deeper layers learn high-level concepts (faces, objects).

04 // Recurrent Neural Network (RNN)

Definition

An RNN is a neural network designed for sequential data where order matters. It maintains a hidden state (memory) that captures information about previous inputs and feeds it back into the network at each time step, allowing the model to process arbitrary-length sequences.

Core Variants

  • Vanilla RNN — Basic recurrent structure; suffers from vanishing/exploding gradients over long sequences.
  • LSTM (Long Short-Term Memory) — Adds input, forget, and output gates to control information flow; solves vanishing gradient for long-range dependencies.
  • GRU (Gated Recurrent Unit) — Simplified LSTM with fewer parameters; uses reset and update gates.
  • Bidirectional RNN — Processes sequences in both forward and backward directions to capture context from both sides.

Key Challenge: Vanishing Gradient

During backpropagation through time (BPTT), gradients must be multiplied across many time steps. If the weights are small, gradients shrink exponentially (vanish) — making it impossible for the model to learn long-range dependencies. LSTMs and GRUs solve this with gating mechanisms that create gradient highways.

05 // Autoencoder

Definition

An autoencoder is an unsupervised neural network that learns to compress data into a low-dimensional latent representation (encoding) and then reconstruct the original data from that representation (decoding). It is trained to minimize reconstruction error between input and output.

Architecture

  • Encoder — Maps high-dimensional input X to a compact latent code Z (bottleneck). Forces the network to learn the most essential features.
  • Bottleneck (Latent Space) — The compressed representation. Its dimensionality is < input; acts as an information bottleneck.
  • Decoder — Reconstructs the original input X̂ from the latent code Z. Mirrors the encoder structure.

Variants (Exam-Critical)

  • Denoising Autoencoder — Trained to reconstruct clean data from corrupted/noisy input; forces more robust feature learning.
  • Variational Autoencoder (VAE) — Encodes input as a probability distribution (mean + variance) rather than a point; enables smooth latent space and new sample generation. This makes it a generative model.
  • Sparse Autoencoder — Adds a sparsity constraint on activations to learn disentangled features.

06 // Transformer

Definition

A Transformer is a neural network architecture built entirely on self-attention mechanisms (no recurrence, no convolution). Introduced in "Attention is All You Need" (2017), it processes all tokens in a sequence in parallel and learns which tokens to attend to via attention weights, enabling highly scalable training on modern hardware.

Core Mechanism: Self-Attention

For each token, the model computes Query (Q), Key (K), and Value (V) vectors. Attention scores are computed as softmax(QKᵀ / √d_k) · V. This lets every token directly attend to every other token in one step — enabling long-range dependencies without the vanishing gradient problem of RNNs.

Key Components

  • Multi-Head Attention — Runs multiple attention mechanisms in parallel, each learning different relationship types.
  • Positional Encoding — Since there's no recurrence, positional info is added explicitly to token embeddings.
  • Feed-Forward Sublayer — Applied position-wise after attention.
  • Layer Normalization & Residual Connections — Stabilize training of deep stacks.
  • Encoder / Decoder stacks — Encoder (BERT-style) for understanding; Decoder (GPT-style) for generation; Both (T5, original Transformer) for seq2seq tasks.

07 // Generative Adversarial Network (GAN)

Definition

A GAN is a generative model consisting of two neural networks trained in opposition: a Generator (G) that learns to produce realistic synthetic data, and a Discriminator (D) that learns to distinguish real data from generated fakes. They are trained simultaneously in a minimax game until the generator fools the discriminator.

Training Dynamic

  • Generator — Takes random noise Z as input and produces synthetic sample G(Z). Tries to maximize the probability that D classifies G(Z) as real.
  • Discriminator — Takes either a real sample or G(Z); outputs probability that input is real. Tries to correctly distinguish real vs. fake.
  • Equilibrium — Training converges (ideally) when the generator produces samples indistinguishable from real data (D outputs 0.5 for all inputs).

Key Challenges (Exam-Critical)

  • Mode Collapse — Generator learns to produce only a few types of outputs, ignoring diversity in the real data distribution.
  • Training Instability — Balancing G and D is difficult; if one overpowers the other, training fails.
  • Vanishing Gradients in D — If D becomes too strong early, G receives near-zero gradients and stops learning.

08 // Applications, Advantages & Disadvantages

Applications by Architecture

ModelApplications
CNNImage classification, object detection (YOLO), medical imaging, facial recognition, self-driving vision, video analysis
RNN/LSTMSpeech recognition, machine translation (pre-Transformer), time-series forecasting, sentiment analysis, music generation
AutoencoderAnomaly detection, data compression, denoising, dimensionality reduction, feature learning, drug discovery
TransformerLarge language models (GPT, BERT), machine translation, summarization, code generation, vision transformers (ViT), multimodal AI
GANImage synthesis/deepfakes, style transfer, data augmentation, super-resolution, video generation, drug molecule design

✓ Advantages

  • Automatically learn hierarchical features from raw data (no manual feature engineering)
  • Scale to massive datasets and compute
  • Achieve superhuman performance on many benchmarks
  • Transfer learning: pre-trained models fine-tuned cheaply
  • Generative models can augment limited datasets
  • Transformers handle long-range dependencies efficiently

✗ Disadvantages

  • Data hungry — require large labeled datasets
  • Computationally expensive to train
  • Black box / poor interpretability
  • Prone to adversarial attacks and dataset bias
  • GANs: training instability and mode collapse
  • RNNs: vanishing gradient, slow sequential training
  • Transformers: quadratic attention complexity (O(n²))
  • Environmental cost of large model training
