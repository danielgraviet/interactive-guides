01

Core Concepts

⚡ Forward Propagation

Forward propagation is the process of passing input data through the neural network layer by layer to produce a prediction (output). Each layer applies weights, biases, and an activation function to transform the signal.

Direction of Flow Input layer → Hidden layer(s) → Output layer. The network uses its current weights to make a prediction — no weight updates happen yet.

Think of it as the network asking "given these weights, what do I predict?"

🔄 Backpropagation

Backpropagation is the algorithm used to update the network's weights after a forward pass. It calculates the gradient of the loss function with respect to each weight, then propagates this error signal backward through the network.

Key Principle Uses the chain rule of calculus to efficiently compute gradients for every weight in every layer. Without backprop, training deep networks would be computationally infeasible.
w_new = w_old − η · (∂L/∂w)
η = learning rate  |  L = loss  |  w = weight

Forward Prop

  • Input → Output prediction
  • Uses current weights
  • Computes the loss
  • Left-to-right through layers
  • No weight modification

Backprop

  • Output → Input (gradient flow)
  • Computes error gradients
  • Updates weights using gradients
  • Right-to-left through layers
  • Modifies weights to reduce loss
Exam Trap Forward prop and backprop always happen in pairs — one forward pass to get predictions and compute loss, then one backward pass to update the weights.
02

MLP Training Steps

🧠 The 4-Step Training Loop

1

Randomly Initialize Weights

All weights in the network are assigned small random values. This breaks symmetry — if all weights started the same, every neuron would learn identical features. Common: Xavier/Glorot initialization or He initialization.

2

Forward Propagation — Make Predictions

Run a batch of training instances through the network. Each neuron computes a weighted sum of its inputs, applies an activation function, and passes the result forward. The final layer produces the prediction ŷ.

3

Compute Loss & Backpropagate

Calculate how wrong the predictions are using a loss function (e.g., MSE for regression, cross-entropy for classification). Backprop computes ∂L/∂w for each weight and uses gradient descent to update them: w ← w − η·∇L.

4

Repeat Until Convergence

Steps 2–3 repeat for every batch, for every epoch, until the loss reaches a minimum (or a stopping criterion is met). Training is complete when the weights have converged to a good solution.

💻 scikit-learn MLP Code

from sklearn.neural_network import MLPClassifier
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

# 1. Pre-process: always scale your features!
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled  = scaler.transform(X_test)

# 2. Define the model
mlpModel = MLPClassifier(
    hidden_layer_sizes=(100, 50),   # 2 hidden layers
    activation='relu',
    learning_rate_init=0.001,
    batch_size=200,
    max_iter=100,               # epochs
    random_state=42
)

# 3. Train
mlpModel.fit(X_train_scaled, y_train)

# 4. Plot loss curve
import matplotlib.pyplot as plt
plt.plot(mlpModel.loss_curve_)
plt.xlabel("Epochs")
plt.ylabel("Loss")
plt.show()
Key Parameters to Know for Exams hidden_layer_sizes sets the architecture. max_iter is the number of epochs. learning_rate_init sets η. batch_size controls mini-batch SGD.
03

Batch Size & Epochs

📦 Batch

A batch is a subset of training instances used in one forward + backward pass. Rather than showing the whole dataset at once (too expensive) or one example at a time (too noisy), we process them in chunks.

Batch Size The number of training instances in a single batch. Typical values: 32, 64, 128, 256, 512.

🔁 Epoch

An epoch is one complete pass through the entire training dataset. During an epoch, every batch is processed exactly once.

Epoch ≠ Training Pass Each epoch contains (dataset_size / batch_size) training passes, not just one.
Batches per epoch = ⌈ N / batch_size ⌉
N = total training instances  |  ⌈⌉ = ceiling function
Total passes = (N / batch_size) × num_epochs
Total forward+backward passes across all training

04

Learning Rate & Gradient Descent

📉 Gradient Descent

Gradient descent is the optimization algorithm used to minimize the loss function. It works by computing the gradient (slope) of the loss with respect to each weight, then moving the weights in the direction that reduces the loss.

w ← w − η · ∇wL(w)
Move each weight opposite to the gradient, scaled by η

The gradient tells you which direction is uphill on the loss surface. Gradient descent moves you downhill to find the minimum.

✓ Small Learning Rate

  • Takes many iterations to converge
  • More likely to find the true minimum
  • Smoother, more stable loss curve
  • Higher computational cost

✗ Large Learning Rate

  • Updates weights quickly
  • May overshoot the minimum
  • Can diverge (loss goes up!)
  • Oscillates around minimum
Convergence vs Divergence A model "converges" when the loss reaches a stable minimum. A model "diverges" when the loss increases or oscillates wildly. The learning rate is the #1 cause of divergence.
05

Improving Training Performance

⚖️

Feature Scaling

Normalize or standardize input features before training. Without scaling, features with large values dominate the gradient updates, causing unstable training. Use StandardScaler (zero mean, unit variance) or MinMaxScaler.

Pre-processing
🔧

Regularization

Controls the magnitude of weights to prevent overfitting. L2 regularization (weight decay) adds a penalty term λ‖w‖² to the loss function. This discourages the model from relying too heavily on any single feature.

Complexity Control
🎲

Dropout

Randomly deactivates a fraction of neurons during each training iteration. This adds structured random noise, forces the network to learn redundant representations, and prevents co-adaptation of neurons — effectively reducing overfitting.

Regularization
🏎️

Adaptive Learning Rates

Algorithms like Adam, RMSProp, and AdaGrad automatically adjust the learning rate per-parameter. Adam combines momentum with adaptive rates and is the most popular optimizer for deep learning today.

Optimization
⏱️

Early Stopping

Monitor validation loss during training and stop when it stops improving. Prevents overfitting by halting training at the point of best generalization, not when training loss is minimized.

Training Control
📐

Batch Normalization

Normalizes the outputs of each layer before passing them to the next. This stabilizes training, allows higher learning rates, and reduces the dependence on careful weight initialization.

Architecture

🎲 Dropout — Deep Dive

Dropout is especially important to understand for exams. During training, each neuron is randomly "dropped" (set to zero) with probability p. At test time, all neurons are active but their outputs are scaled by (1−p) to compensate.

TRAINING

Each iteration, randomly zero-out nodes with probability p. Forces the network to use multiple pathways — can't rely on one set of neurons.

INFERENCE

All neurons are active. Outputs are multiplied by (1−p) to keep expected values the same as during training (inverted dropout).

Why it Prevents Overfitting By randomly disabling neurons, dropout prevents any single neuron from becoming overly specialized to the training data. The model must learn distributed, robust representations.

QUESTION 01 A neural network is trained on 80,000 instances with a batch size of 400 and 50 epochs. How many total weight update steps (training passes) are completed?
A 50
B 4,000
C 10,000
D 200,000
✓ Correct! 80,000 / 400 = 200 batches per epoch. 200 × 50 = 10,000 total training passes. Each batch = one weight update step.
QUESTION 02 Which of the following best describes what happens when the learning rate is set too high?
A The model converges faster and achieves a lower loss
B Training slows down significantly due to more iterations needed
C The weights may overshoot the minimum, causing the loss to oscillate or diverge
D Dropout automatically compensates by increasing regularization
✓ Correct! A large η makes weight updates too big — the optimizer "jumps over" the minimum and oscillates. In severe cases, the loss increases every iteration (divergence). B describes a small LR. D is unrelated.
QUESTION 03 What is the PRIMARY role of backpropagation in training a neural network?
A To generate predictions from input data
B To compute gradients of the loss with respect to each weight so they can be updated
C To normalize the input features before training
D To randomly initialize the weights at the start of training
✓ Correct! Backprop's job is to compute ∂L/∂w for every weight using the chain rule. These gradients are then used by gradient descent to update the weights. A describes forward propagation; C is pre-processing; D is weight initialization.
QUESTION 04 You notice your model's training loss is decreasing steadily, but the validation loss starts increasing after epoch 20. Which technique would MOST directly address this problem?
A Increase the learning rate
B Increase the batch size
C Apply dropout or regularization, and use early stopping
D Remove feature scaling
✓ Correct! This is the classic sign of overfitting — the model has memorized training data but generalizes poorly. Dropout, regularization (L2), and early stopping are the standard remedies. Increasing LR or batch size would likely make it worse.
QUESTION 05 In scikit-learn's MLPClassifier, the parameter max_iter=100 refers to what?
A The maximum number of weight update steps
B The maximum number of epochs (full passes through the training set)
C The maximum batch size
D The maximum number of hidden layers allowed
✓ Correct! max_iter in sklearn's MLP is the number of epochs. The total weight updates = (dataset_size / batch_size) × max_iter.
QUESTION 06 Which statement about dropout is CORRECT?
A Dropout is applied during inference to reduce prediction variance
B Dropout permanently removes neurons from the network
C Dropout randomly deactivates neurons each iteration during training, helping prevent overfitting
D Dropout is a form of feature scaling
✓ Correct! Dropout is a training-time regularization technique. It randomly zeros neurons per iteration. At inference/test time, all neurons are active (outputs scaled by 1−p). It does NOT permanently remove neurons.
QUESTION 07 A model trained without feature scaling on a dataset where one feature ranges from 0–1 and another ranges from 0–10,000. What is the most likely outcome?
A The model will train normally; feature scale has no effect on neural networks
B The large-valued feature will dominate gradient updates, causing unstable or slow training
C The model will automatically normalize features during backpropagation
D The learning rate will automatically adjust to compensate
✓ Correct! Without scaling, the large-magnitude feature (0–10,000) will produce much larger gradients than the small one (0–1). This causes imbalanced updates — the network struggles to learn from the small-scale feature, often failing to converge properly.
QUESTION 08 What does the mlpModel.loss_curve_ attribute store in scikit-learn?
A The loss value computed on the test set after training
B The training loss value at the end of each epoch
C The loss value for each individual training instance
D The gradient values at the final epoch
✓ Correct! loss_curve_ stores the loss computed on the training data at the end of each epoch. It has length = number of epochs and is used to diagnose convergence, learning rate issues, and overfitting.

What-If Scenarios

Click each scenario to reveal deep analysis. These are the kinds of questions that appear on hard exams.

WHAT IF You double the batch size from 100 → 200, keeping everything else the same
›
Original setup: 10,000 training instances, batch size 100, 50 epochs, LR = 0.001. You change batch size to 200.

Effect on Batches Per Epoch

Batches per epoch: 10,000/100 = 100 → 10,000/200 = 50. You now process half as many batches each epoch.

Effect on Total Weight Updates

Total passes: 100 × 50 = 5,000 → 50 × 50 = 2,500. You perform HALF the weight updates in the same number of epochs.

Effect on Gradient Quality

Larger batches give a more accurate (less noisy) estimate of the true gradient, but the noisiness of small batches can help escape local minima and often generalizes better.

Effect on Training Speed

Each epoch completes faster (fewer backward passes), but the model may need more epochs to converge to the same quality since each epoch makes fewer updates.

Effect on Generalization

Research shows larger batches often converge to sharper minima that generalize worse ("sharp minima" problem). Smaller batches tend to find flatter minima that generalize better.

Bottom Line Doubling batch size ≈ halving the effective training time per epoch and reducing update noise, but may hurt generalization. To compensate, many practitioners also scale the learning rate proportionally (linear scaling rule: if batch doubles, try doubling LR too).
WHAT IF The loss curve oscillates wildly and never decreases
›
You train a MLP for 100 epochs. The loss_curve_ plot shows the loss bouncing between high values rather than trending downward.

Most Likely Cause

The learning rate is too large. The gradient updates are overshooting the minimum on every step — the optimizer jumps back and forth over the optimal weights.

Secondary Causes

  • Features are not scaled — large gradient magnitudes cause instability
  • Batch size is too small — extremely noisy gradient estimates
  • Poor weight initialization (rare with modern defaults)
  • The loss function is poorly suited to the problem

How to Diagnose

Start by reducing learning rate by 10× (e.g., 0.01 → 0.001). If still unstable, check whether features are scaled. Examine the magnitude of gradients.

How to Fix

  • Reduce learning rate (primary fix)
  • Apply StandardScaler to features
  • Increase batch size to get smoother gradient estimates
  • Use Adam optimizer instead of vanilla SGD
WHAT IF Training loss is very low but test accuracy is poor
›
After 200 epochs, your model achieves 99% training accuracy but only 62% test accuracy. The loss_curve_ shows a smooth decrease to near zero.

Diagnosis

This is textbook overfitting. The model has memorized the training data rather than learning generalizable patterns. The large gap between training and test performance is the key indicator.

Why It Happens

  • Model is too complex (too many parameters) for the dataset size
  • Too many epochs — model has "memorized" training noise
  • No regularization to constrain weight magnitudes

Solutions

  • Dropout: Randomly deactivate neurons during training to prevent co-adaptation
  • L2 Regularization: Add penalty to loss for large weights (α parameter in sklearn)
  • Early Stopping: Stop training when validation loss starts increasing
  • More data: Overfit on more diverse examples
  • Reduce model complexity: Fewer hidden layers or neurons
WHAT IF You increase epochs from 50 → 500, but loss barely improves after epoch 80
›
Your loss_curve_ shows steep descent for the first 80 epochs, then essentially flatlines. You continue training to 500 epochs.

What's Happening

The model has already converged (or reached a local minimum) by epoch 80. The extra 420 epochs are mostly wasted computation — and worse, could cause overfitting.

Risks of Over-training

  • Overfitting: the model continues to fit training noise after convergence
  • Wasted compute time with no benefit
  • Potential degradation in test performance

Better Approaches

  • Early stopping: Monitor validation loss and stop at the optimal point
  • Learning rate decay: Reduce LR over time to make finer adjustments once near the minimum
  • Use the loss curve: mlpModel.loss_curve_ to visually identify convergence
Rule of Thumb If the loss hasn't improved by more than ~0.001 in 10 consecutive epochs, the model has likely converged. This is exactly what scikit-learn's tolerance parameter (tol) controls.
WHAT IF You remove StandardScaler before training your MLP
›
Your dataset has features with ranges: age (18–80), salary (20,000–200,000), and score (0.0–1.0). You skip feature scaling and train directly.

Immediate Effect on Gradients

The salary feature (range ~180,000) produces gradients ~10,000× larger than the score feature (range ~1). Gradient descent will update weights connected to salary much more aggressively than those connected to score.

Effect on Convergence

The loss surface becomes elongated and poorly scaled — gradient descent takes a very zig-zaggy path instead of going straight to the minimum. Convergence is much slower or fails entirely.

Effect on Weight Initialization

Weights are initialized assuming ~unit-scale inputs. With unscaled inputs, the network's activations may immediately saturate or become extremely small, hampering learning.

Fix

Always use StandardScaler().fit_transform(X_train) and .transform(X_test). Fit ONLY on training data to prevent data leakage.

Common Exam Mistake Calling fit_transform on the test set instead of just transform is data leakage — the scaler learns test statistics, inflating performance estimates.
